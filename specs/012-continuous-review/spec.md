# Feature Specification: 復習の連続プレイ

**Feature Branch**: `012-continuous-review`
**Created**: 2026-07-13
**Status**: Draft
**Input**: User description: "復習が終わった後に、連続で復習をしたい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 結果画面から続けて復習する (Priority: P1)

復習セッション（1バッチ、最大20件）を終えたユーザーが、期日を迎えている項目がまだ残っている場合、結果画面から直接次のバッチのプレイを開始できる。ダッシュボードに戻って改めて「復習」を選び直す必要がない。

**Why this priority**: 現状は1バッチ（最大20件）で強制的に打ち切られ、続けるにはダッシュボードへの往復が必要。復習対象が多いユーザーほど摩擦が大きく、これが解消されないと本機能の価値が実現しない中核体験。

**Independent Test**: 期日超過の復習項目を21件以上持つユーザーで検証する。1バッチ目（20件）を終えた結果画面に「続けて復習する」旨のアクションが表示され、選択すると新しいバッチのプレイが（ダッシュボードを経由せず）直接始まることを確認する。

**Acceptance Scenarios**:

1. **Given** 期日を迎えている復習項目が21件以上ある状態で1バッチ（20件）のプレイを終えた、**When** 結果画面を見る、**Then** 「続けて復習する」旨のアクションが表示される。
2. **Given** 結果画面に「続けて復習する」旨のアクションが表示されている、**When** それを選択する、**Then** ダッシュボードに戻ることなく次のバッチのプレイが直接始まる。
3. **Given** 続けて開始した新しいバッチのプレイを終えた、**When** 結果画面を見る、**Then** その新しいバッチのみの正答率・苦手一覧が表示される（前のバッチの成績と合算されない）。

---

### User Story 2 - 残り件数を見て続けるか判断する (Priority: P2)

ユーザーは結果画面上で、期日を迎えている残りの項目がどれくらいあるかを確認したうえで、続けるかどうかを判断できる。

**Why this priority**: 続行アクションの存在自体はP1で満たされるが、残り件数が分からないと「あと少しで終わるのか、まだ大量にあるのか」を把握できず、続けるかの意思決定がしづらい。UXの質を高める付随機能。

**Independent Test**: 期日を迎えている復習項目の残数が既知の状態（例: 13件）でバッチを終え、結果画面のアクションにその残数（13件）が表示されることを確認する。また、残数の取得に失敗した場合でも「続けて復習する」アクション自体（件数表示なし）は表示され、続行できることを確認する。

**Acceptance Scenarios**:

1. **Given** バッチ終了時点で期日を迎えている残りの項目が13件ある、**When** 結果画面を見る、**Then** 「続けて復習する」旨のアクションに残数13件が表示される。
2. **Given** 残数の取得ができなかった（一時的な取得失敗）、**When** 結果画面を見る、**Then** 件数表示のない「続けて復習する」旨のアクションが表示される（続行できること自体は残数取得の成否に依存しない）。

---

### Edge Cases

- 続けて開始したバッチの終了時点で、期日を迎えている項目がすでに0件になっていた場合 → 「続けて復習する」旨のアクションは表示されず、既存の「今日の復習はありません」という完了状態が表示される。
- 続行時に取得した新しいバッチが（重複除去などの結果）実質0件だった場合 → 同様に既存の「今日の復習はありません」の完了状態にフォールバックする。
- 残り件数の表示後、実際に続行ボタンを押すまでの間に他デバイスで同じ項目が消化されていた場合 → 表示していた残数と実際の対象件数が食い違うことを許容する（厳密なリアルタイム一致は求めない）。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは、1バッチの復習プレイが終わった時点で、そのユーザーについて期日を迎えている復習項目がまだ存在するかを判定できなければならない。
- **FR-002**: 期日を迎えている項目が残っている場合、結果画面に「続けて復習する」旨のアクションを表示しなければならない。
- **FR-003**: 「続けて復習する」旨のアクションを選択した場合、ダッシュボードなど他の画面に遷移することなく、次のバッチのプレイを直接開始しなければならない。
- **FR-004**: 続行後の新しいバッチは、初回バッチと同じ選定条件（期日を迎えている項目からの、最大20件・ランダム選定）に従わなければならない。
- **FR-005**: 各バッチの結果表示（正答率・苦手一覧）は、そのバッチ単体の成績のみを表示しなければならない。複数バッチを連続でプレイしても、成績を合算・累積してはならない。
- **FR-006**: 期日を迎えている項目が残っていないと判明した場合（残数の取得に成功し、かつ0件であった場合）、「続けて復習する」旨のアクションを表示してはならず、既存の完了体験（ダッシュボードへ／おすすめクイズを試す）をそのまま維持しなければならない。
- **FR-007**: 残り件数の判定に失敗した場合でも、システムは復習の完了自体をブロックせず、「続けて復習する」旨のアクション自体は表示しなければならない（件数表示のみを省略する）。続行アクションを非表示にするのは、残数が確実に0件であると判明した場合（FR-006）に限る。実際に続けた結果0件だった場合はFR-008のフォールバックに従う。
- **FR-008**: 続行して取得した新しいバッチが結果的に0件だった場合、システムは既存の「今日の復習はありません」という完了状態を表示しなければならない。
- **FR-009**: 「続けて復習する」旨のアクションには、期日を迎えている残りの件数を表示しなければならない（表示時点での件数であり、その後の状況変化との厳密な一致は求めない）。ただし残数の取得に失敗した場合は、件数表示を省略し、アクション自体はFR-007に従って表示する。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 期日を迎えている復習項目を21件以上持つユーザーが、ダッシュボードへ戻ることなく2バッチ目以降のプレイを開始できる。
- **SC-002**: 結果画面に表示される残り件数は、表示された時点での実際の残り件数と一致する（同一デバイス・同一セッション内での通常操作において）。
- **SC-003**: 期日を迎えている項目がちょうど0件になった時点でバッチを終えたユーザーには（残数取得に成功した通常時）、通常の結果画面（正答率・苦手一覧＋おすすめクイズ導線／ダッシュボード導線）がそのまま表示され、続行アクションは表示されない。「今日の復習はありません」の完了状態は、バッチ開始前の時点で既に対象項目が0件だった場合（初回ロード時、または続行後の新バッチが実質0件だった場合＝FR-008）にのみ表示される、別の画面状態である。

## Assumptions

- 期日を迎えている復習項目の選定ロジック（ランダム順、1バッチ最大20件という上限）は変更せず、続行時のバッチにもそのまま適用する。
- 複数バッチにまたがる成績の累積・セッション全体のサマリー表示は本機能のスコープ外とする（各バッチは独立して結果表示される）。
- 本機能は間隔反復の復習フロー（期日を迎えた項目の復習）のみを対象とし、他のクイズモード（都道府県クイズ、おすすめクイズの都度プレイなど）には影響しない。
- 間隔反復のスケジューリングアルゴリズム自体（次回復習日の計算方法）に変更は加えない。
