# Feature Specification: 今日のおすすめクイズにおける地域選択（絞り込み）機能の改善

**Feature Branch**: `011-recommend-region-filter`  
**Created**: 2026-07-10  
**Status**: Draft  
**Input**: User description: "今日のおすすめクイズにおける地域選択（絞り込み）機能の改善。今日のおすすめクイズを開始する前に、特定の地域（特定の地方や都道府県）に絞り込んで出題できるようにする。現行は「除外する地方」をトグルするUIはあるが、ポジティブに「この地方や都道府県だけをプレイする」といった選択や、都道府県単位での絞り込みには対応していない。"

## 背景 / 問題

「今日のおすすめクイズ」は学習履歴（復習時期、苦手傾向など）を元に出題を自動推薦する機能である。
現在、ユーザーはクイズ開始前の調整ダイアログ（`RecommendOverride`）でクイズモードや出題数を変更できるが、地域については「除外する地方」を指定するネガティブなフィルタリングしかできない。
「今回は東北地方の市区町村だけを練習したい」「大阪府と京都府に絞ってプレイしたい」といった、特定の地域（地方または都道府県）を能動的に指定（ポジティブ選択）して出題プールを絞り込むニーズに対応していない。
また、都道府県単位での細かい絞り込みにも対応していないため、広範な地方の中の特定の県だけにフォーカスして学習することが困難である。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 地方単位でのポジティブな絞り込み (Priority: P1)

ユーザーは「今日のおすすめクイズ」の調整ダイアログで、出題対象としたい「地方」（例：「東北」「関東」など、複数選択可能）を選択して開始できる。
「全国」が選ばれている、あるいはどの地方も選択されていない場合は、全国の市区町村が出題対象となる。

**Why this priority**: 本機能の最も基本的な要件。特定の地方に絞ったクイズの作成を容易にする。

**Independent Test**:
調整ダイアログで「東北」のみを選択して開始する。出題されたクイズの市区町村が、すべて東北地方の県のものであることを確認する。

**Acceptance Scenarios**:

1. **Given** 調整ダイアログが開いている状態、 **When** 「東北」のみを対象として選択しクイズを開始する、 **Then** 出題されるすべての市区町村の `region` が「東北」であること。
2. **Given** 調整ダイアログが開いている状態、 **When** 「全国」を選択（または全地方を未選択に）してクイズを開始する、 **Then** 全国の市区町村から出題されること。

---

### User Story 2 - 都道府県単位でのポジティブな絞り込み (Priority: P2)

ユーザーは地方だけでなく、さらに細かく「都道府県」単位で対象地域を選択（複数選択可能。例：「宮城県」「福島県」）してクイズを開始できる。
地方を選択すると、その地方に属する都道府県がUI上に展開され、個別にオン/オフを切り替えられる。

**Why this priority**: ユーザーがさらに狭い範囲（特定の県）に集中してクイズを解くための学習効率向上に直結する。

**Independent Test**:
調整ダイアログで「東北」を選択し、展開された都道府県リストから「宮城県」と「福島県」のみを有効にして開始する。出題される市区町村が「宮城県」または「福島県」のいずれかに属していることを確認する。

**Acceptance Scenarios**:

1. **Given** 調整ダイアログで「東北」が選択されている状態、 **When** 「宮城県」と「福島県」のみをオンにして開始する、 **Then** 出題されるすべての市区町村の `prefecture` が「宮城県」または「福島県」のいずれかであること。
2. **Given** 地方「北海道」が選択されている状態、 **When** 「北海道」をオフにする、 **Then** 自動的に属する「北海道」（都道府県）も選択から解除されること。

---

### User Story 3 - 設定の永続化と通常クイズ設定への連動 (Priority: P3)

ユーザーが今日のおすすめクイズで設定した「対象地方」「対象都道府県」はブラウザ（LocalStorage）に保存され、次回ダイアログを開いた際にも維持される。また、必要に応じて通常クイズ画面（`/quiz/municipality/[mode]`）の設定とも連動する。

**Why this priority**: 毎回同じ絞り込みを行う手間を省き、UXを向上させる。

**Independent Test**:
「東北」の「宮城県」を選択した状態でクイズを開始し、結果画面からダッシュボードに戻り、再度おすすめクイズのダイアログを開く。選択状態が維持されていることを確認する。

**Acceptance Scenarios**:

1. **Given** 調整ダイアログで地域を選択した状態、 **When** クイズを開始して再度ダッシュボードに戻りダイアログを開く、 **Then** 前回選択した地域（地方・都道府県）が選択された状態で表示されること。

---

### User Story 4 - 制覇率に基づく難易度ステップアップ制限 (Priority: P2)

正答率が高くても、現在挑戦している最高難易度セルの市区町村制覇率（全市区町村に対する「過去正解したことがある市区町村」の割合）が 90% 未満の場合、おすすめクイズで次の難易度（例: easyからmedium）へ自動的にステップアップして提案されることを防ぐ。これにより、簡単な問題がまだ数多く残っている段階で急激に高難易度の問題に移行するのを防ぐ。

**Why this priority**: ユーザーの学習体験を最適化し、完璧主義的な学習フロー（各難易度をしっかりと制覇してから次へ進む）を支えるために重要。

**Independent Test**:
easy 難易度の正答率移動平均が 100% だが、制覇率（coverageRate）が 50% の状態でおすすめクイズの推薦を取得した際、提案される難易度が `easy` のままであり、`medium` に進行しないことを検証する。

**Acceptance Scenarios**:

1. **Given** ユーザーの easy 難易度の正答率平均が 80% 超、 **When** easy 難易度の制覇率が 50% である場合、 **Then** 推薦される難易度リストに `medium` が含まれず `easy` のままであること。
2. **Given** ユーザーの easy 難易度の正答率平均が 80% 超、 **When** easy 難易度の制覇率が 90% 以上である場合、 **Then** 推薦される難易度が `medium` にステップアップすること。

### Edge Cases

- **選択した地域の市区町村プールが不足している場合**:
  - 選択された地域（地方/都道府県）と難易度の組み合わせにより、出題プール内の市区町村数が要求された出題数（10/20/30問）に満たない場合、開始ボタンを非活性（Disabled）にするか、あるいはプール内の全件（例: 6問のみ）でクイズを開始するか。
  - **仕様判断**: クイズ開始に必要な最低問題数に満たない場合、開始ボタンを非活性とし「対象の市区町村が足りません」等の警告を表示する。ただし、1問以上あればプレイ開始可能とするか、あるいは設定された問題数に満たない場合は警告しつつ、存在する全件で開始することを許容する（通常クイズ画面の挙動と揃える）。
- **全ての都道府県のチェックを外した場合**:
  - 全てのチェックが外れた場合は「全国」が選択されたものとみなし、出題制限を設けない。

## Requirements *(mandatory)*

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 「今日のおすすめクイズ」調整ダイアログ（`RecommendOverride`）において、地方（北海道、東北、関東、中部、近畿、中国、四国、九州）をポジティブにトグル選択できるUIを提供する。
- **FR-002**: 各地方の選択状態に応じて、その地方に属する「都道府県」をトグル選択できるサブUI（ドロップダウンまたはアコーディオンなど、モバイル表示を損なわない形式）を提供する。
- **FR-003**: 選択された地方・都道府県の情報をURLクエリパラメータ（`region` および `prefectures`）にシリアライズし、クイズプレイ画面（`/quiz/municipality/[mode]`）に遷移させる。
- **FR-004**: クイズプレイ画面の設定初期化処理および `buildQuestions` ロジックにおいて、`prefectures` パラメータが指定されている場合は、その都道府県に属する市区町村のみを出題対象としてフィルタリングする。
- **FR-005**: 選択した条件で出題プール（市区町村件数）が0件となる場合、クイズの開始ボタンを非活性化し、適切なエラーラベルを表示する。
- **FR-006**: 選択された地域フィルタは LocalStorage に保存され、ダイアログ再表示時に復元される。
- **FR-007**: 推薦エンジン（`evaluateProgression`）での難易度進行（ステップアップ）判定において、現在の最高難易度のFitZoneセルに該当する `cellCoverages` の平均制覇率が 90% 未満の場合、難易度進行をロックする。


### Key Entities

- **地域フィルタ設定 (RegionFilterSettings)**:
  - `targetRegions`: 選択された地方名の配列（例: `['東北', '関東']`）
  - `targetPrefectures`: 選択された都道府県名の配列（例: `['青森県', '宮城県', '東京都']`）
- **クイズ設定 (Settings)**:
  - 既存の `Settings` エンティティを拡張し、`prefectures: string[]` プロパティを追加する。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーは今日のおすすめクイズで、特定の地方のみに絞り込んだクイズを2タップ以内で開始できる。
- **SC-002**: ユーザーは特定の都道府県のみ（複数可）に絞り込んだクイズを開始でき、出題される問題がすべて指定の都道府県のものである（正答/誤答を問わず）。
- **SC-003**: 地域絞り込み適用によるクイズ開始時のクエリパフォーマンス悪化がなく、遷移后1秒以内に出題の第1問が表示される。

## Assumptions

- **モバイル最優先**: 地域および都道府県の選択UIは、モバイル幅（375px）のボトムシート内でスクロールやレイアウト崩れを起こさないよう、コンパクトかつ折りたたみ可能な設計とする。
- **データ互換性**: 既存の `getRecommendation` Server Action 自体には地域フィルタの絞り込み機能を持たせず、クライアントサイドでの出題生成ロジック（`buildQuestions`）でのフィルタリングにて地域制限を実現する（既存の推薦ロジック・推薦キャッシュを壊さないため）。
