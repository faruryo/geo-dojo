# 契約定義書 (Contracts): 地域絞り込み機能のコンポーネント・URL契約

## 1. URL クエリパラメータ契約

今日のおすすめクイズ（`RecommendContent`）からクイズページ（`/quiz/municipality/[mode]`）へ遷移する際、以下のクエリパラメータを引き渡します。

| パラメータ | 役割 | 形式 | 例 |
|---|---|---|---|
| `region` | 選択された地方のフィルタ | カンマ区切りの文字列 | `region=東北,関東` |

### 整合性ルール
- クイズ開始時のパースロジックは、無効な地方名（`REGIONS` に含まれない値）を無視しなければなりません。
- 「全国」がクエリパラメータに含まれている場合、または指定がない場合は全地方を出題対象とします。

---

## 2. UI 状態連携契約 (LocalStorage)

調整ダイアログの表示状態および永続化データは以下の構造に従います。

### キー名: `geodojo-recommend-region-filters`

### 値の定義
```typescript
interface SavedRegionFilters {
  targetRegions: string[]; // 選択された地方名（例: ['東北']、空の場合は全国）
}
```

### ライフサイクル仕様
- **マウント時 (Read)**:
  - ローカルストレージから `SavedRegionFilters` を読み取る。
  - 値が存在する場合、`RecommendOverride` コンポーネントおよびクイズ設定の初期状態に適用する。
  - 値が存在しない、あるいは破損している場合は、デフォルト値（`targetRegions = []` ＝ 全国）でフォールバックし、全地域を対象とする。
- **変更・開始時 (Write)**:
  - ユーザーが地方をトグル選択した際、現在の状態をシリアライズして LocalStorage に即座に保存する。
