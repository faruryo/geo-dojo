## テスト実行

```bash
pnpm test                     # Vitest（追加したフィルタロジックなどのテスト実行）
pnpm lint                     # 型チェック / Lint
```

## 動作確認（ローカル）

### 準備

```bash
supabase start                # ローカル Supabase（起動していない場合のみ）
pnpm dev
```

ブラウザで `http://127.0.0.1:3000` を開く。

> [!NOTE]
> **ログインについて**  
> ローカル環境ではメール確認（Email Confirmation）が無効化されています。アカウントが存在しない、あるいはログインできない場合は、新規アカウント作成画面 (`/signup`) から適当なメールアドレスとパスワード（6文字以上）を入力して登録してください。確認メールの受信を待つことなく即座にアカウントが作成され、ログインできます。

### 動作確認手順

#### 1. 地方単位での絞り込み確認
1. ダッシュボードから「今日のおすすめクイズ」の「設定（ギアアイコン）」または調整領域を開く。
2. 地方のトグルボタン（例: 「東北」）を選択し、他を未選択にする。
3. 「そのまま開始」をクリック。
4. 開始されたクイズをプレイし、出題される市区町村がすべて「青森県」「岩手県」「宮城県」「秋田県」「山形県」「福島県」のいずれかであることを確認する。

#### 2. 単一都道府県ガードの確認（モードA/B）
1. 調整ダイアログでモードを「A（逆引き地図）」または「B（逆引き4択）」に設定し、地方は「北海道」のみを選択する。
2. 「そのまま開始」ボタンが非活性化され、「対象地方を2つ以上選んでください」等の警告が表示されることを確認する。
3. 地方の選択を「北海道」「東北」の2つ以上に変更すると、ボタンが活性化されクイズを開始できることを確認する。

#### 3. 出題プール不足時の挙動確認
1. 選択した地方・難易度の組み合わせで出題プールが要求数に満たないケース（例: 狭い地方＋高難易度）を作り、要求数に満たない全件でクイズが正常に開始されることを確認する。

#### 4. 設定の永続化確認
1. 調整ダイアログで「近畿」を選択してクイズを開始。
2. 解答途中、または結果画面からダッシュボードに戻る。
3. 再度「今日のおすすめクイズ」の調整ダイアログを開いた際、前回の選択肢（「近畿」）が選択された状態で復元されていることを確認する。

#### 5. 制覇率に基づく難易度ステップアップ制限の確認
1. easy 難易度の正答率移動平均が高い（80%超）が、easy 難易度セルの制覇率（coverageRate）が 90% 未満のユーザー状態で「今日のおすすめクイズ」の推薦を取得する。
2. 提案される難易度が `medium` にステップアップせず `easy` のまま維持されることを確認する（`__tests__/lib/quiz/recommendation-progression.test.ts` の自動テストで代替可）。
