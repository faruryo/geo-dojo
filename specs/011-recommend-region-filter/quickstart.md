# Quickstart: 今日のおすすめクイズにおける地域選択（絞り込み）機能の改善

## テスト実行

```bash
pnpm test                     # Vitest（追加したフィルタロジックなどのテスト実行）
pnpm lint                     # 型チェック / Lint
```

## 動作確認（ローカル）

### 準備

```bash
supabase start                # ローカル Supabase（起動していない場合のみ）
pnpm dev
```

ブラウザで `http://127.0.0.1:3000` を開く。

> [!NOTE]
> **ログインについて**  
> ローカル環境ではメール確認（Email Confirmation）が無効化されています。アカウントが存在しない、あるいはログインできない場合は、新規アカウント作成画面 (`/signup`) から適当なメールアドレスとパスワード（6文字以上）を入力して登録してください。確認メールの受信を待つことなく即座にアカウントが作成され、ログインできます。

### 動作確認手順

#### 1. 地方単位での絞り込み確認
1. ダッシュボードから「今日のおすすめクイズ」の「設定（ギアアイコン）」または調整領域を開く。
2. 地方のトグルボタン（例: 「東北」）を選択し、他を未選択にする。
3. 都道府県はデフォルト（すべてオン）の状態で「そのまま開始」をクリック。
4. 開始されたクイズをプレイし、出題される市区町村がすべて「青森県」「岩手県」「宮城県」「秋田県」「山形県」「福島県」のいずれかであることを確認する。

#### 2. 都道府県単位での絞り込み確認
1. 調整ダイアログで「東北」を選択する。
2. 「東北」の下部に展開された都道府県リストから、「宮城県」と「福島県」以外のチェック（トグル）を外す。
3. クイズを開始し、出題される市区町村が「宮城県」または「福島県」のみであることを確認する。

#### 3. エラー・境界ケースの確認
1. 調整ダイアログで特定の都道府県のみを選択し、かつクイズプレイ設定で該当市区町村の難易度（例: 「達人」）を選択したとき、プールサイズが0になる組み合わせ（例: 「徳島県」で「達人」問題が存在しないなど）を作る。
2. 開始ボタンが非活性化され、スタートできない状態（「該当する市区町村なし」等の警告）になることを確認する。

#### 4. 設定の永続化確認
1. 調整ダイアログで「近畿」→「京都府」「大阪府」を選択してクイズを開始。
2. 解答途中、または結果画面からダッシュボードに戻る。
3. 再度「今日のおすすめクイズ」の調整ダイアログを開いた際、前回の選択肢（「近畿」と「京都府」「大阪府」）が選択された状態で復元されていることを確認する。
