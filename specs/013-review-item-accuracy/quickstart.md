# Quickstart: 復習項目一覧の正答率表示

## テスト実行

```bash
pnpm test                     # Vitest
DATABASE_URL=... pnpm test    # getItemAccuracyData の DB 統合テスト込み（queries-parity.test.ts 等）
pnpm lint                     # 型チェック / Lint
```

## 動作確認（ローカル）

### 準備

```bash
supabase start                # ローカル Supabase（起動していない場合のみ）
supabase db reset             # 必要なら（マスタ再投入含む）
pnpm tsx scripts/sync-municipality-master.ts   # municipality_master が空の場合
pnpm dev
```

ブラウザで `http://127.0.0.1:3000` を開く。

> [!NOTE]
> **ログインについて**
> ローカル環境ではメール確認（Email Confirmation）が無効化されています。アカウントが存在しない、あるいはログインできない場合は、新規アカウント作成画面 (`/signup`) から適当なメールアドレスとパスワード（6文字以上）を入力して登録してください。確認メールの受信を待つことなく即座にアカウントが作成され、ログインできます。

### 事前準備: 正答率にばらつきのある復習項目を作る

複数の市区町村クイズをプレイし、一部の市区町村で意図的に正解・不正解を混ぜて解答する（例: ある市区町村×モードは5回中4回正解、別の市区町村×モードは5回中1回正解）。`srsRecords` が `status='reviewing'` になっている（＝復習中）ことを確認する。

### 動作確認手順

#### 1. 正答率の表示（US1 / FR-001〜004）
1. `/quiz/review/items` を開く。
2. 一覧の各行に正答率（パーセンテージ）が表示されることを確認する。
3. 事前に5回中4回正解させた項目の正答率が「80%」と表示されることを確認する（Studio の SQL Editor で `municipality_quiz_results` を直接確認し、実際の正解数/総数と一致するかも照合する）。
4. 同じ市区町村を複数モードで復習中にしている場合、モードごとに別々の正答率が表示され、合算されていないことを確認する。

#### 2. 低正答率の視覚的区別（US2 / FR-007）
1. 正答率が大きく異なる項目（例: 30%・80%）が同一ページに並ぶ状態で一覧を開く。
2. 正答率の低い項目が、色などで他の項目と視覚的に区別できることを確認する。

#### 3. 正答率取得失敗時のフォールバック（FR-006）
1. Chrome DevTools の Network タブ「Request blocking」機能、または `getItemAccuracyData` の呼び出しに一時的に `throw` を仕込んでエラーを再現する。
2. その状態で `/quiz/review/items` を開く。
3. 正答率の表示のみが省略され、種目バッジ・市区町村名・次回復習日は通常どおり表示されること、一覧のページング操作が引き続き可能であることを確認する。

#### 4. 試行回数が少ない項目（Edge Case）
1. ある項目を1回だけ解答した状態（正解または不正解）にする。
2. その項目の正答率が0%または100%としてそのまま表示され、非表示や特別な注記が付かないことを確認する。
