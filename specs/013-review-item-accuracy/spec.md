# Feature Specification: 復習項目一覧の正答率表示

**Feature Branch**: `013-review-item-accuracy`
**Created**: 2026-07-14
**Status**: Draft
**Input**: User description: "覚えている途中の市区町村一覧ページ（/quiz/review/items）で、各項目（市区町村×モード）ごとの正答率を表示する"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 項目ごとの正答率を確認する (Priority: P1)

「覚えている途中の市区町村」一覧ページを開いたユーザーは、そこに並ぶ各項目（市区町村×モードの組）について、自分がこれまでどれくらい正解できているか（正答率）を確認できる。

**Why this priority**: 現状の一覧には種目バッジと次回復習日しか表示されておらず、どの項目が得意でどの項目が苦手なのかが分からない。正答率が見えて初めて、このページが「復習の進み具合を振り返る場」として機能する。これが本機能の中核価値。

**Independent Test**: 復習中の項目（srsRecords が status='reviewing' の項目）を複数持つユーザーで一覧ページを開き、各行に正答率（パーセンテージ）が表示され、その値が当該市区町村×モードに対するこれまでの解答試行の正解割合と一致することを確認する。

**Acceptance Scenarios**:

1. **Given** 復習中の項目が一覧に表示されている、**When** 一覧ページを開く、**Then** 各項目の行に正答率がパーセンテージで表示される。
2. **Given** ある項目（市区町村×モード）に対してこれまで5回解答し4回正解している、**When** 一覧ページを見る、**Then** その項目の正答率として80%が表示される。
3. **Given** 同じ市区町村が複数のモードで復習中である、**When** 一覧ページを見る、**Then** モードごとに別々の正答率が表示される（合算されない）。

---

### User Story 2 - 正答率が低い項目を一目で見分ける (Priority: P2)

ユーザーは一覧の中で正答率が低い項目を視覚的にすぐ見分けられ、どの項目を優先して復習すべきか判断しやすくなる。

**Why this priority**: 正答率の数値自体はP1で見えるようになるが、項目数が多い一覧の中で低い数値を目で探すのは負担が大きい。視覚的な強調があることで、一覧の実用性が高まる。

**Independent Test**: 正答率が異なる複数の項目（例: 30%・60%・90%）が並ぶ一覧で、正答率が低い項目が他と視覚的に区別できる状態（強調表示）になっていることを確認する。

**Acceptance Scenarios**:

1. **Given** 正答率が大きく異なる項目が複数一覧にある、**When** 一覧ページを見る、**Then** 正答率が低い項目は他の項目と視覚的に区別できる。

---

### Edge Cases

- ある項目の解答試行がこれまで1回しかない場合 → その1回の結果に基づく正答率（0%または100%）がそのまま表示される（試行回数が少ないことを理由に非表示にはしない）。
- 正答率データの取得に失敗した場合 → その項目の正答率表示のみを省略し、種目バッジや次回復習日など他の情報は通常どおり表示する。一覧全体の表示がブロックされてはならない。
- 一覧に表示される項目自体が0件（復習中の項目がない）の場合 → 既存の空状態表示のまま変更しない。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは、「覚えている途中の市区町村」一覧ページ（/quiz/review/items）に表示される各項目（市区町村×モードの組）について、その項目の正答率を表示しなければならない。
- **FR-002**: 正答率は、当該市区町村×モードの組に対するこれまでの全解答試行のうち正解した試行の割合として算出しなければならない。
- **FR-003**: 正答率はパーセンテージ（例: 80%）として表示しなければならない。
- **FR-004**: 同一市区町村が複数モードで復習中の場合、正答率はモードごとに個別に算出・表示しなければならず、モード間で合算してはならない。
- **FR-005**: 正答率の表示は、既存の「解答内容（都道府県名など）を一覧に表示しない」という制約（流暢性の錯覚回避のため）を損なってはならない。正答率は成績指標であり、解答そのものではないため、都道府県名等は引き続き非表示のままとする。
- **FR-006**: 正答率の取得に失敗した場合でも、システムは当該項目の他の表示（種目バッジ・次回復習日）をブロックしてはならず、正答率表示のみを省略しなければならない。
- **FR-007**: システムは、一覧内で正答率が低い項目を他の項目と視覚的に区別できる形で表示しなければならない。

### Key Entities *(include if feature involves data)*

- **復習項目（市区町村×モード）**: ユーザーが現在復習中（未卒業）の、市区町村と出題モードの組。一覧ページの各行に対応する。
- **解答履歴**: 各復習項目に対する過去の解答試行の記録（正解/不正解）。正答率はこの履歴から算出される。
- **正答率**: ある復習項目について、解答履歴に基づき算出される「正解試行数 ÷ 総試行数」の割合。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 「覚えている途中の市区町村」一覧ページを開いたユーザーは、追加の操作なしに各項目の正答率を確認できる。
- **SC-002**: 表示される正答率は、その項目に対するこれまでの全解答試行から算出した値と常に一致する。
- **SC-003**: 正答率データの取得に失敗しても、一覧ページの表示（他の項目・他の情報）は通常どおり保たれる。
- **SC-004**: 正答率が低い項目とそうでない項目を、ユーザーが一覧を一目見ただけで区別できる。

## Assumptions

- 正答率は、当該市区町村×モードに対するこれまでの全解答試行（復習中になってからの試行に限らず、初回学習時の試行も含む）を対象に算出する。既存の「苦手ランキング」機能（ダッシュボード）と同じ集計範囲・方式を踏襲する。
- 正答率表示の対象は「覚えている途中の市区町村」一覧ページ（/quiz/review/items）に限定し、クイズのプレイ画面・結果画面など他の画面の表示や挙動には影響を与えない。
- 解答内容（都道府県名等）を一覧で非表示にする既存方針（005-spaced-review, FR-016a）は変更しない。正答率という集計値の追加のみがスコープであり、解答そのものを推測できる情報を新たに追加するものではない。
- 「正答率が低い」項目を強調する際の具体的な閾値や表現方法（色分け・アイコン等）は実装時に決定してよく、本仕様では「視覚的に区別できること」のみを要件とする。
- 試行回数が極端に少ない項目（1〜2回）についても、正答率はそのまま実際の値を表示する。試行回数不足を理由に表示を隠したり、注記を追加したりすることはスコープ外とする。
