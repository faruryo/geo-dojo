### User Story 5 - 市区町村クイズ (Priority: P5)

ユーザーが複数のゲームモードで市区町村クイズを行い、日本全国の市区町村の位置・所属県を習得する。

**Why this priority**: 都道府県クイズの次のステップとして、より細かい地理知識を問う。4つのモードと地域フィルター・問題数設定により、習熟度に合わせた学習が可能。

**Independent Test**: `/quiz/municipality` にアクセスし、1問以上出題されて正誤フィードバックが表示されれば独立して機能する。

**Game Modes**:
- **モードA（逆引き地図）**: 市区町村名 → 日本地図で所属都道府県をタップ。同名市区町村は1問にまとめる（`m.name` でユニーク化）
- **モードB（逆引き4択）**: 市区町村名 → 4択ボタンで都道府県を選択。**ディストラクターは選択地域内の他都道府県から抽選**（地域内 < 4 都道府県の場合は全国フォールバック）。出題プールは (name, prefecture) で重複排除（政令市区が同名で重複出題されないように）
- **モードC（順引き4択）**: 都道府県名 → 4択から正解市区町村を選択。**ディストラクターも選択地域内の他都道府県から抽選**（地域内 < 4 都道府県の場合は全国フォールバック）。ターゲット都道府県と同名の市区町村名を除外する（中央区など同名重複対策）。出題プールは (name, prefecture) で重複排除
- **モードD（順引き地図）**: 市区町村名 → 都道府県にズームインした地図上でタップ。**1問あたり30秒の制限時間**。タイムアウトまたは初回不正解は同じ問題で1回だけ再挑戦できる。政令市の区は e-Stat 正式名（例: 仙台市青葉区）で個別エントリとして出題し、重複排除は行わない（各区が別ポリゴン）

**Settings**:
- 地域フィルター：全国 / 地方（8区分）の **複数選択（chip）**。'全国' は全地域選択ショートカット。個別地域を選ぶと '全国' は外れ、全て解除すると '全国' に自動復帰
- 1セッション問題数：10（デフォルト）/ 20 / 30問
- 苦手優先モード：過去の不正解を優先出題
- Mode B は選択地域の合計都道府県数が 2 未満の場合（北海道のみ選択時）は自動無効化し、UI ボタンを disabled 表示

**Acceptance Scenarios**:

1. **Given** `/quiz/municipality` を開くと、**When** モード・地域・問題数を設定して開始すると、
   **Then** 設定に従った市区町村が出題される。
2. **Given** モードAで「府中市」が出題されたとき、**When** ユーザーが東京都と広島県をタップして「解答する」を押すと、
   **Then** 両方正解として処理され、次の問題に進む（1か所のみでは不正解）。
3. **Given** モードAで同名市区町村が出題されたとき、**When** 問題画面が表示されると、
   **Then** 「N か所あります」（N = ユニークな都道府県数）と正解箇所の総数が事前に表示され、選択中は「あと M か所」の残数カウンターが更新される。N=1 の場合は表示しない（同一都道府県内に複数エントリがある政令市区などの誤表示を防ぐため）。
4. **Given** モードAで都道府県を選択した状態で、**When** 同じ都道府県を再タップすると、
   **Then** 選択が解除される（誤タップのやり直しが可能）。「解答する」ボタンを押すまでは確定しない。
5. **Given** モードAで「解答する」ボタンが押されたとき、**When** 選択箇所が正解箇所と完全一致しなかった場合、
   **Then** その問題は不正解として記録され、正解箇所がハイライト表示される（不足・過剰どちらも不正解）。
6. **Given** クイズが終了したとき、**When** 結果画面が表示されると、
   **Then** 正解数・正答率・苦手な市区町村一覧が表示される。
7. **Given** クイズ結果が記録されたとき、**When** 次回セッションで「苦手優先」をオンにすると、
   **Then** 過去に不正解だった市区町村が優先的に出題される。

### User Story 6 - 市区町村クイズの難易度フィルター (Priority: P6)

ユーザーが難易度バケット（☆〜☆☆☆☆）で市区町村プールを絞り込み、自分のレベルに合った問題で学習する。
村や無名な町を除外したカジュアルプレイから、上級者向けの全難易度プレイまでをカバーする。

**Why this priority**: 既存US5の補完。「村は難しすぎる」というカジュアル層の参入障壁を下げる。

**Independent Test**: `/quiz/municipality` 設定画面で難易度☆のみ選択 → 開始 → 出題されたすべての市区町村が☆バケットに属することを確認。

**Difficulty Tiers**:
- 段階導入: **Phase 1** は市/町/村種別ベース、**Phase 2** は e-Stat 国勢調査の人口データベースへ自動切替（バッチ実行で `municipality_master.difficulty` を更新）
- Phase 1（種別ベース）:
  - ☆ 入門: 政令指定都市の区
  - ☆☆ 中級: 市
  - ☆☆☆ 上級: 町
  - ☆☆☆☆ 達人: 村
- Phase 2（人口ベース）:
  - ☆ 入門: 人口 ≥ 100,000
  - ☆☆ 中級: 30,000〜99,999
  - ☆☆☆ 上級: 10,000〜29,999
  - ☆☆☆☆ 達人: < 10,000

**Settings**:
- 難易度フィルター: chip 形式の複数選択。デフォルト [☆, ☆☆]
- 地域フィルター AND 難易度フィルター で出題プール決定
- 全難易度未選択時はスタートボタン無効化

**Acceptance Scenarios**:

1. **Given** ☆ のみ選択して開始すると、**When** 問題が出題されると、
   **Then** 全問題が☆バケットの市区町村である。
2. **Given** [☆☆☆, ☆☆☆☆] を選択し関東を選んで開始すると、**When** 問題が出題されると、
   **Then** 関東地方の町・村のみが出題される。
3. **Given** 難易度・地域の組み合わせでプールが0件のとき、**When** スタートを押すと、
   **Then** エラーメッセージが表示され開始しない。
4. **Given** 全難易度のチェックを外したとき、**When** 設定画面を見ると、
   **Then** スタートボタンが無効化されている。
5. **Given** 苦手優先モードと難易度☆☆を組み合わせると、**When** 出題されると、
   **Then** ☆☆プール内で苦手市区町村が優先される。
6. **Given** Phase 2 完了後に e-Stat バッチを再実行すると、**When** 国勢調査データが更新されると、
   **Then** `municipality_master.difficulty` バケットが人口に応じて再計算される。

### Edge Cases

- 地図クイズでネットワーク切断時の動作 — 地図データはオフラインキャッシュから表示可能とする。
  ただし採点結果の保存はオンライン復帰後に同期する（部分的オフライン対応、完全オフライン動作はMVP外）。
- 市区町村クイズで同名市区町村（例：府中市＝東京都・広島県）が出題された場合、全箇所をタップするまで「解答する」ボタンを無効化し、残数を表示する。
- 市区町村クイズのモードDで市区町村境界TopoJSONのロードに失敗した場合、モードCへ自動フォールバックしてエラーを表示する。
- 市区町村クイズで難易度＋地域フィルターの組み合わせが0件のとき、エラーメッセージを表示しスタートさせない。
- `municipality_master` テーブルがまだ初期化されていない（バッチ未実行）とき、難易度フィルターUIをグレーアウトし「データ準備中」を表示する。
- 地域フィルターの全選択を解除した場合は '全国' に自動復帰させる（空状態を許容しない）。
- 北海道のみを選択した状態で Mode B を選んでいた場合、自動で Mode C に切り替える（Mode B ボタンは disabled で「この地域では選択不可」と表示）。
- プールが設定問題数より少ない場合は「該当 N 件のみ — 毎回同じ問題が繰り返されます。難易度か地域を広げてください」と警告を表示する（出題自体は許可、技術的な重複排除はしない）。
- 政令市の区（仙台市青葉区など）は e-Stat 正式名で個別エントリとして扱う。Mode A/B/C では同じ都道府県を答える問題が複数回プールに入りうるが、これは現状の許容範囲（将来的に高難易度トグルでの集約を検討、T095）。

- **FR-031**: 市区町村クイズは2段階の URL 構造を取る:
  - `/quiz/municipality` = モード選択画面（A/B/C/D + 各モードの説明と白地図ベースのミニプレビュー）
  - `/quiz/municipality/[mode]` = 詳細設定（地域・問題数・難易度）+ プレイ + 結果
  
  プレイ中のブラウザバック（および「中断」リンク）はモード選択画面に戻り、プレイ状態は破棄する。
- **FR-032**: URL によってモードが固定されているため、設定画面ではモードを切り替えられない。Mode B で地域選択が 2 都道府県未満（北海道のみ等）になった場合は自動切替せず、スタートボタンを無効化して「地域を追加してください」と表示する（FR-025 の自動切替を撤回）。

### Key Entities *(include if feature involves data)*

- **市区町村クイズ記録**: 市区町村クイズの正解/不正解履歴。municipality_name（市区町村名）・prefecture（都道府県）・is_correct・mode（A/B/C/D）・answered_at を持つ。ユーザーが所有する。苦手優先モードの出題重み付けに使用される。
- **市区町村マスタ**: 全市区町村の静的＋派生メタデータ。`code`（団体コード5桁、主キー）・`name`・`prefecture`・`region`・`population`（nullable）・`population_year`・`difficulty`（`easy`/`medium`/`hard`/`expert`）・`updated_at` を持つ。e-Stat API からのバッチ取り込みで人口を更新し、難易度バケットを再計算する。読み取りは認証ユーザー共通（個人データではないが、未認証アクセスは許可しない）。

## Assumptions

- ユーザーはアカウント登録・ログイン済みの状態から各機能を使用する（認証フローはMVPスコープに含むが、本仕様の範囲外とする）。

- 都道府県クイズの正解判定は都道府県単位。市区町村クイズはモードにより都道府県単位（モードA/B）または市区町村単位（モードC/D）で判定する。

## Clarifications

### Session 2026-05-09

- Q: 同名市区町村（複数県に存在）のタップ方式は？ → A: タップで選択状態にし「解答する」ボタンで送信（やり直し可）。未選択箇所がある間は残数カウンターを表示し「解答する」ボタンを無効化する。
- Q: 市区町村クイズの結果をDBに保存するか？ → A: 保存する（苦手優先モードに活用）。municipality_quiz_results テーブルを新設。
- Q: モードD（順引き地図）を初期リリースに含めるか？ → A: 含める。`public/japan-municipalities.topojson` を生成して同梱する。
- Q: 1セッションのデフォルト問題数は？ → A: 10問（既存の都道府県クイズと統一）。設定で10/20/30問から選択可能。
- Q: 市区町村クイズは認証必須か？ → A: 認証必須（既存の `app/(app)` グループに配置）。

### Session 2026-05-23

- Q: 難易度の信号は？ → A: Phase 1 は市/町/村種別、Phase 2 は人口（e-Stat 国勢調査 2020）。Phase 3 のクラウド正答率は MVP 外。
- Q: 人口データのソースとアーキテクチャは？ → A: e-Stat API + 専用バッチスクリプトで `municipality_master` テーブルに upsert（静的 JSON ではなく DB 管理）。
- Q: 難易度フィルターの選択方式は？ → A: chip 形式の複数選択。デフォルト [☆, ☆☆]。全選択解除でスタート無効化。
- Q: 既存の `municipality_quiz_results` テーブルとの関係は？ → A: 直交（独立）。`municipality_master.code` は論理的に参照されるが DB 制約は付けない（master 更新時の整合性は code が変わらないことに依存）。
- Q: バッチの実行先は？ → A: **ローカル手動のみ**（`pnpm tsx scripts/sync-municipality-master.ts`）。国勢調査は5年に1回しか更新されないため、Vercel Cron や Supabase pg_cron 等の自動スケジューラは MVP 外。本番デプロイ後に必要性が出たら別 spec で追加する。

### Session 2026-05-24

- Q: Mode B で地域を選んでも全国の都道府県が選択肢に出るのは正しい？ → A: 誤り。**選択地域内の他都道府県** からディストラクターを抽選する（FR-026）。地域 < 4 都道府県は全国フォールバック。
- Q: 北海道（1 都道府県のみ）で Mode B はどう扱う？ → A: 選択不可（disabled）。地域変更時に Mode B から Mode C へ自動切替（FR-025）。
- Q: 地域フィルターは複数選択にする？ → A: する。chip 形式の multi-select、'全国' は select-all ショートカット。全解除で '全国' 自動復帰（FR-024）。
- Q: Mode C のディストラクター市区町村も地域内制約するか？ → A: する。Mode B と同じ方針（FR-026）。
- Q: 名古屋市16区など同名で複数エントリある政令市区は Mode B で重複出題されてしまう。どう扱う？ → A: Mode B/C のプールを (name, prefecture) で重複排除（FR-027）。Mode D は個別ポリゴンなので排除しない。
- Q: 仙台市5区などのトポJSON `nam_ja` が「仙台市」になっており Mode D でどの区をタップすべきか分からない。どう解決？ → A: e-Stat `getMetaInfo` の正式名（仙台市青葉区など）を `sync-municipality-master.ts` で取得し DB / `municipalities.json` を上書き（FR-028）。
- Q: Mode A の「N か所あります」が政令市区の重複で誤表示される（例: 仙台市で5か所）。どう修正？ → A: N = `correctPrefectures.size`（ユニーク都道府県数）で計算し、N=1 の場合は表示しない（Acceptance Scenario 3 更新済）。
- Q: 入門×北海道など小プールで同じ問題が繰り返される。どう対応？ → A: 警告文を強化するのみ（FR-029）。セッション間重複排除や難易度自動拡張は実装しない。
- Q: 政令市の区を Mode A/B でも個別出題することで「同じ都道府県を答える問題」が複数回出るが対応する？ → A: 現状の許容範囲。将来 T095 で高難易度モード（区単位で区別を学ぶ）への切替トグルを別 spec で検討。
- Q: `/quiz` を開くといきなり都道府県クイズが始まり、市区町村クイズ中はブラウザバックで都道府県クイズに戻ってしまう。どう整理する？ → A: 3段階に分離（FR-031）: `/quiz` = クイズ種類選択（都道府県 / 市区町村）、`/quiz/prefecture` = 都道府県クイズ、`/quiz/municipality` = 市区町村クイズのモード選択、`/quiz/municipality/[mode]` = 詳細設定+プレイ。プレイ中の中断・ブラウザバックは常にモード選択画面に戻る。
- Q: モード選択画面の UX は？ → A: 4 モードカード（A/B/C/D）の右側（モバイルは下）に説明文＋白地図ベースの実 UI ミニプレビューを表示。Mode A/D は topojson 白地図で該当都道府県をハイライト、Mode B/C はダミーデータの 4 択 UI を表示。プレビュー領域は `min-h-[30rem]` で固定し、選択モード切替時に「進む」ボタンの位置が変わらないようにする。
- Q: モードDの制限時間 15 秒が短すぎる。どうする？ → A: **30 秒**に延長（FR-030 更新）。初回不正解または時間切れは不正解として記録するが、同じ問題で 1 回だけ再挑戦できる挙動は維持。
- Q: Mode B が地域変更時に Mode C へ自動切替される挙動は妥当？ → A: 妥当ではない（モードが URL でロックされたため）。FR-025 の自動切替を撤回し、Mode B で地域 < 2 都道府県時は警告＋スタート無効化のみとする（FR-032）。
