# Feature Specification: GeoDojo MVP（Phase 1）

**Feature Branch**: `001-geodojo-mvp`
**Created**: 2026-05-06
**Status**: Draft
**Input**: User description: "GeoDojoのMVP（Phase 1）— SRS学習、地図タップクイズ、カード作成、AI生成レビュー"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - SRS学習セッション (Priority: P1)

GeoGuessrを学習したいユーザーが、フラッシュカードを使って間隔反復学習（SRS）セッションを行い、
日本の地理的特徴（看板の文字、電柱のスタイル、道路標識など）を効率的に記憶定着させる。

**Why this priority**: プラットフォームのコアバリュー。このストーリーなしに他の機能は意味をなさない。
SRSフィードバックループが機能すれば、他機能は独立した追加機能として扱える。

**Independent Test**: カードが1枚存在する状態でユーザーが評価（1/3/5）を行い、
次回出題日が正しく更新されることを確認するだけで独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** 学習カードが1枚以上ある状態で、**When** ユーザーが「学習を始める」を選択すると、
   **Then** 当日期限のカードが優先的に表示される。
2. **Given** カードが表示されている状態で、**When** ユーザーが「全然（1）」「うろ覚え（3）」「完璧（5）」
   のいずれかを選択すると、**Then** 評価に応じた次回出題日が設定される（低評価ほど短い間隔）。
3. **Given** カードに視覚的なマーカーが設定されている場合、**When** カードが表示されると、
   **Then** 画像上の正確な位置にマーカーが重ねて表示される（画面サイズに関わらず位置がズレない）。
4. **Given** 当日期限のカードがない状態で、**When** ユーザーが学習画面を開くと、
   **Then** 次回出題予定日と「今日の学習は完了です」メッセージが表示される。

---

### User Story 2 - 地図タップクイズ (Priority: P2)

ユーザーが日本地図を見て、出題された都道府県をタップして位置を当てるクイズを行い、
地理的な位置感覚を養う。

**Why this priority**: SRS学習で記憶した特徴を地図上の位置と結びつける実戦的な確認手段。
P1の後に独立して追加可能。

**Independent Test**: 日本地図が表示され、都道府県を1つタップし正誤判定が表示されれば単独で機能する。

**Acceptance Scenarios**:

1. **Given** ユーザーが地図クイズを開始すると、**When** 問題が出題されると、
   **Then** 日本の全47都道府県が識別可能な形で地図上に表示される。
2. **Given** 問題が出題された状態で、**When** ユーザーが都道府県をタップすると、
   **Then** 正解の場合は緑色、不正解の場合は赤色でフィードバックが即座に表示され、
   正解の都道府県名が示される。
3. **Given** クイズが終了したとき、**When** 結果画面が表示されると、
   **Then** 正解数・正答率・苦手な都道府県一覧が表示される。

---

### User Story 3 - 手動カード作成 (Priority: P3)

ユーザーがGeoGuessrプレイ中に気づいた地理的特徴（看板の文字、道路の形状など）を
スクリーンショットとメモで記録し、学習カードとして保存する。

**Why this priority**: ユーザー自身の発見を即座にカード化できることで、学習の自発性を高める。
P1が機能すればカードの追加方法として独立して追加可能。

**Independent Test**: 画像をアップロードし、テキストを入力してカードを保存できれば独立して機能する。

**Acceptance Scenarios**:

1. **Given** カード作成画面で、**When** ユーザーが画像ファイルをアップロードすると、
   **Then** 画像がプレビュー表示され、テキスト入力欄が有効になる。
2. **Given** 画像が表示された状態で、**When** ユーザーが画像上の特定箇所をタップすると、
   **Then** その位置に視覚的なマーカーが追加され、ラベルを入力できる。
3. **Given** 内容が入力された状態で、**When** 「カードを保存」を押すと、
   **Then** カードが自分のデッキに追加され、次の学習セッションから出題対象になる。
4. **Given** 未対応の画像形式またはサイズ上限（10MB）超過の場合、**When** アップロードすると、
   **Then** わかりやすいエラーメッセージが表示され、再試行できる。

---

### User Story 4 - AI生成カード候補のレビュー (Priority: P4)

ユーザーがAIによって自動生成されたカード候補（画像の特徴説明付き）を確認し、
承認・編集・却下することで効率的に学習デッキを充実させる。

**Why this priority**: 手動作成を補完するAI生成により、学習コンテンツを迅速に拡充できる。
P3の後に追加可能なHITL（人間が最終判断）フロー。

**Independent Test**: AI候補が1件表示され、承認または却下できれば独立して機能する。

**Acceptance Scenarios**:

1. **Given** AI生成処理が完了した状態で、**When** ユーザーがレビュー画面を開くと、
   **Then** AI生成カード候補一覧（画像・提案テキスト付き）が表示される。
2. **Given** 候補が表示された状態で、**When** ユーザーが「承認」を選択すると、
   **Then** そのカードがデッキに追加され、以後のSRS学習で出題される。
3. **Given** 候補が表示された状態で、**When** ユーザーが「却下」を選択すると、
   **Then** 候補は削除され、学習データには含まれない。
4. **Given** 候補が表示された状態で、**When** ユーザーがテキストを編集して「承認」を選択すると、
   **Then** 編集内容が反映されたカードが追加される。
5. **Given** AI生成処理が進行中の場合、**When** ユーザーがレビュー画面を開くと、
   **Then** 処理中であることを示すステータスが表示される（空白画面ではない）。

---

### Edge Cases

- 学習デッキが空のとき、SRS学習セッションを開始しようとした場合の動作（ガイダンス表示）。
- 地図クイズでネットワーク切断時の動作 — 地図データはオフラインキャッシュから表示可能とする。
  ただし採点結果の保存はオンライン復帰後に同期する（部分的オフライン対応、完全オフライン動作はMVP外）。
- AI生成が失敗またはタイムアウトした場合のフォールバック動作（エラー通知・再試行）。
- 複数デバイス間での学習データの同期タイミング。
- カード作成時に画像に個人情報（顔・車のナンバー等）が含まれる場合の扱い（ユーザーの自己責任を明示）。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムはユーザーの評価（1段階/3段階/5段階）を記録し、SM-2ベースの間隔反復アルゴリズムで次回出題日を計算しなければならない。
- **FR-002**: システムはカードの視覚的マーカーを、あらゆる画面サイズで画像に対して正確な相対位置に表示しなければならない（ピクセル座標ではなく相対座標で管理）。
- **FR-003**: システムは日本の全47都道府県をタップ可能な形で地図上に表示し、選択した都道府県名を識別しなければならない。
- **FR-004**: ユーザーは画像をアップロードし、テキストメモと視覚的なマーカーを付けて学習カードを作成できなければならない（JPEG/PNG、最大10MB）。
- **FR-005**: システムはAI生成カード候補をユーザーが承認するまで学習デッキに追加してはならない（HITLプロセスの強制）。
- **FR-006**: ユーザーは自分のカードをタグで絞り込んで一覧表示・削除できなければならない。
  システムタグ（都道府県・地方・特徴カテゴリ）での絞り込みと、ユーザー任意タグの追加・削除が可能であること。
- **FR-007**: すべての学習データへのアクセスには認証が必要であり、未ログインユーザーはログイン画面にリダイレクトされなければならない。
- **FR-008**: システムはAI生成処理の進行状況（処理中・完了・失敗）をユーザーに通知しなければならない。
- **FR-010**: システムはカードに対してシステムタグ（都道府県・地方・特徴カテゴリ）を自動付与しなければならない。
- **FR-011**: 地図クイズ用の地図データはオフライン環境でも表示可能なよう事前にキャッシュされなければならない。
  採点記録はオンライン復帰後に自動同期されること。
- **FR-009**: 地図クイズの結果（正解/不正解/都道府県名）は即座（タップから1秒以内）にフィードバックされなければならない。

### Key Entities *(include if feature involves data)*

- **フラッシュカード**: 学習の基本単位。画像参照情報、テキストメモ（特徴説明）、視覚的マーカーのリストを持つ。ユーザーが所有する。
- **視覚的マーカー（アノテーション）**: カード画像上の特定位置に配置されたラベル付きマーカー。画像サイズに対する相対座標（0.0〜1.0）で位置を表現する。
- **SRS記録**: 特定カードに対するユーザーの学習履歴。最終評価スコア、次回出題予定日、累計学習回数を持つ。
- **タグ**: カードに付与する属性ラベル（例：「東北」「木製電柱」「お気に入り」）。
  1枚のカードに複数のタグを設定できる。タグによるフィルタリングが「デッキ」の役割を果たす。
  - **システムタグ（自動）**: 都道府県・地方（東北・関東など）・特徴カテゴリ（電柱・看板など）が自動付与される。
  - **ユーザータグ（手動）**: ユーザーが任意のタグを追加・削除できる。
  - ユーザーは「デッキを作成する」という操作を意識せずに、タグで学習範囲を絞り込んで学習できる。
- **AI生成候補**: AIが提案したカード草稿。ユーザーが承認するまで「仮」状態を保ち、承認後にフラッシュカードとなる。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーはアプリを開いてから最初のフラッシュカードを評価するまで30秒以内に完了できる。
- **SC-002**: 都道府県をタップしてから正誤フィードバックが表示されるまでの待機時間が1秒未満である。
- **SC-003**: ユーザーはスクリーンショットのアップロードからカード保存完了まで3分以内に完了できる。
- **SC-004**: AI生成カード候補のレビューにより、手動カード作成と比較して1枚あたりの所要時間が50%以上短縮される。
- **SC-005**: 幅375px（スマートフォン基準）の画面において、全主要操作（カード評価・地図タップ・カード保存・候補承認）が1本指タップのみで完結できる。
- **SC-006**: SRSアルゴリズムの精度として、「完璧（5）」と評価されたカードの次回出題間隔が連続評価のたびに延長される動作が100%のケースで確認できる。
- **SC-007**: 初回使用のユーザーが機能説明なしにSRS学習セッションを完了できる割合が80%以上である（ユーザーテスト基準）。

## Assumptions

- ユーザーはアカウント登録・ログイン済みの状態から各機能を使用する（認証フローはMVPスコープに含むが、本仕様の範囲外とする）。
- 学習対象は日本国内の地理特徴のみ（海外地域はMVP対象外）。
- 地図クイズの正解判定は都道府県単位（市区町村レベルの精度は不要）。
- AI生成パイプラインは非同期処理とし、ユーザーはリアルタイムの完了を待つ必要はない（完了後に通知）。
- MVPのターゲットユーザーはGeoGuessrを既にプレイしており、地理学習への意欲がある日本語話者。
- カード上の画像はユーザーが自身でアップロードしたもの（GeoGuessrのスクリーンショット）に限定し、Street Viewのパノラマ画像本体は保存しない。
- デスクトップ対応はMVPで「壊れない」程度に対応するが、最適化はモバイルを優先する。
