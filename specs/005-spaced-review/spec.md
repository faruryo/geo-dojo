# Feature Specification: 科学的間隔反復による間違い復習

**Feature Branch**: `005-spaced-review`
**Created**: 2026-06-01
**Status**: Draft
**Input**: User description: "一度間違えた問題を復習する機能を作りたい。復習おすすめのあたりから移動する感じかな。科学的な復習、アクティブリコール、間隔反復等の科学的理論に則って復習できるような機能を検討して欲しい"

## 概要

学習者が「一度間違えた市区町村」を、推測ではなく記憶研究で確立された手法（**アクティブリコール** = 自力で答えを想起する／**間隔反復** = 復習間隔を最適な日数に空ける）に基づいて復習できるようにする。

現状の「復習おすすめ」は *誤答あり × 最終回答から7日以上経過 × 誤答率順* という固定ルールで、間隔は学習者の定着度に適応しない。本機能はこれを **回答結果に応じて1問ごとに次回復習日を動的に算出する間隔反復方式（SM-2）** へ置き換え、ダッシュボードの「復習おすすめ」付近を入口とする。

## Clarifications

### Session 2026-06-01

- Q: 導入前に既に間違えている履歴を、復習対象に取り込むか？ → A: 既存の誤答ログから復習対象を初期生成（バックフィル）し、すぐ期日到来（導入直後から復習可能）にする
- Q: 復習セッションでの回答を既存のクイズ回答ログに記録するか？ → A: 通常どおりログに記録する（既存の正答率推移・苦手ランキング等に反映される）
- Q: SM-2 の回答の質(0–5)を正誤二値だけで決めるか、時間/ヒントも反映するか？ → A: 二値のみ（不正解=2／正解=4）。回答時間・ヒントによる補正はしない（ヒント機能は存在しない）
- Q: 「定着済み」になった問題はその後どう扱うか？ → A: 卒業＝復習スケジュールから外し以後は出題しない。十分な間隔＋連続正解で定着済みとし、再び通常クイズで誤答したら復習対象に復帰する
- Q: 復習セッション以外（通常クイズ）で復習対象に正解したとき、SM-2 を前進させるか？ → A: 前進させる（どのクイズの正解でも間隔拡大・卒業に反映。同日重複の反映上限などは plan で調整）
- Q: 誤答した問題を当日中に復習出題するか？ → A: しない（翌日以降のまま）。答えを見た直後の再出題は間隔効果・望ましい困難の観点で長期定着に寄与しないため、科学的妥当性を優先。代わりに「覚えている途中のアイテム一覧」をメタ認知/進捗として表示する（答え＝都道府県は伏せ、流暢性の錯覚を避ける）

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 期日が来た間違いを復習する (Priority: P1)

学習者は、過去に間違えた市区町村のうち「今日復習すべき」ものだけを出題され、自力で答えを思い出して回答する。回答の結果に応じて、その問題の次回復習日が自動的に再スケジュールされる（正解できれば間隔が広がり、間違えれば間隔がリセットされて短期間で再出題される）。

**Why this priority**: これが機能の核（間違いの登録 → 期日管理 → 復習実施 → 再スケジュール）であり、この1本だけで「科学的に間違いを復習できる」という価値が完結する。他のストーリーはこの体験を補助・可視化するもの。

**Independent Test**: 通常クイズでわざと数問間違える → 一定の経過後にダッシュボードの「今日の復習」から復習を開始 → 期日が来た問題のみが出題され、正解した問題の次回期日が延び、間違えた問題は短期で再度期日が来ることを確認できる。

**Acceptance Scenarios**:

1. **Given** 学習者が通常クイズである市区町村×モードを初めて誤答した, **When** 回答が確定する, **Then** その (市区町村, モード) が復習対象として登録され、初回の復習期日が設定される
2. **Given** 復習対象に期日到来済みの問題が存在する, **When** 学習者が「今日の復習」を開始する, **Then** 期日が現在以前の問題だけが（期日が古い順に）出題される
3. **Given** 復習問題が出題されている, **When** 学習者が自力で回答して正解する, **Then** その問題の連続正解回数が増え、次回復習期日が前回より長い間隔で再設定される
4. **Given** 復習問題が出題されている, **When** 学習者が誤答する, **Then** その問題の連続正解回数がリセットされ、短い間隔（翌日相当）で次回期日が再設定される
5. **Given** 期日到来済みの復習問題が0件, **When** 学習者がダッシュボードを見る, **Then** 「今日の復習はありません」と表示され、復習開始導線は次回予定日を案内する

---

### User Story 2 - ダッシュボードから今日の復習量を把握して着手する (Priority: P2)

学習者は、ダッシュボードの「復習おすすめ」付近で「今日復習すべき件数」を一目で把握し、ワンタップで復習セッションに入れる。

**Why this priority**: 復習行動の継続には「今日やるべき量が見える」ことが重要。ただし P1 の復習ループが動けば最低限の入口は成立するため、可視化の充実は次点。

**Independent Test**: 復習対象を複数登録し期日を到来させた状態でダッシュボードを開くと、正しい件数バッジと開始ボタンが表示され、押すと期日到来分のみの復習セッションが始まることを確認できる。

**Acceptance Scenarios**:

1. **Given** 期日到来済みの復習問題が N 件ある, **When** 学習者がダッシュボードを開く, **Then** 「今日の復習 N件」が表示される
2. **Given** 「今日の復習」カードが表示されている, **When** 学習者が開始ボタンを押す, **Then** 期日到来分のみで構成された復習セッションへ遷移する
3. **Given** 1回の復習セッションで出題できる上限を超える件数が期日到来している, **When** セッションを開始する, **Then** 期日が古い／定着度が低い順に上限件数まで出題され、残りは次セッションに持ち越される

---

### User Story 3 - 復習の進捗と定着度を確認する (Priority: P3)

学習者は、自分の間違いがどれだけ復習・定着できているか（定着済み件数、今後の復習予定）を確認でき、学習のモチベーションを保てる。

**Why this priority**: 継続率・満足度を高める価値はあるが、復習そのものは P1/P2 で完結するため最後。

**Independent Test**: 複数問題を異なる定着段階まで復習した状態で進捗表示を開き、定着済み件数・今後7日間の復習予定件数が実データと一致することを確認できる。

**Acceptance Scenarios**:

1. **Given** 復習対象に様々な定着段階の問題がある, **When** 学習者が進捗を確認する, **Then** 「復習中」「定着済み」の件数が区別して表示される
2. **Given** 将来の復習期日を持つ問題がある, **When** 学習者が進捗を確認する, **Then** 今後の復習予定（例: 今後7日間の日別件数）が確認できる
3. **Given** ある問題が十分な間隔で連続正解し定着条件を満たした, **When** その問題が定着済みになる, **Then** 「今日の復習」件数からは除外され、定着済みとして集計される

---

### Edge Cases

- **同名市区町村**: 異なる都道府県の同名市区町村は別の復習対象として区別される（既存のコード単位の扱いに準拠）。Mode A（同名を1問にまとめる出題）での復習対象の単位は本機能の設計事項として明確化する（Assumptions 参照）。
- **モード差**: 同じ市区町村でも出題モードが異なれば別の復習対象として独立にスケジュールされる（例: 渋谷区×Mode A と 渋谷区×Mode D は別管理）。
- **時間制限モード（Mode D）**: 制限時間内に正解できなかった／タイムアウトした場合は誤答として扱う。
- **回答の質は二値固定**: SM-2 の質は不正解=2／正解=4 のみ（回答時間やヒントによる補正なし。ヒント機能は存在しない）。Mode D のタイムアウト/制限時間内未正解は誤答（質2）として扱う。
- **長期未ログイン後の大量期日到来**: 期日を過ぎた問題が大量に溜まっても、セッション上限と優先順位（期日が古い／定着度が低い順）で過負荷にならないようにする。
- **既存の誤答履歴（本機能導入前のデータ）**: 既に蓄積された誤答ログを復習対象としてどう扱うかを明確化する（Assumptions 参照）。
- **モード混在セッション**: 1セッションは複数モードが混在しうる（A/B/C/D が入り混じる）。地図選択 → 4択 → 地図タップ等、問題ごとに設問UIが切り替わっても破綻なく連続して回答できること。途中の操作モード変化が誤操作・誤答を誘発しないUI配慮が必要。
- **同一問題が同一セッション内で期日更新**: 1セッション中に同じ問題を重複出題しない。

## Requirements *(mandatory)*

### Functional Requirements

**間違いの捕捉と登録**

- **FR-001**: システムは、学習者がクイズ（通常クイズ・復習セッションを含む）で市区町村を誤答したとき、その (市区町村, 出題モード) を一意の復習対象として登録しなければならない（未登録の場合）。
- **FR-001a**: システムは、本機能導入時に既存の誤答ログから (市区町村, 出題モード) 単位の復習対象を初期生成（バックフィル）し、初回期日を即時到来として設定しなければならない。これにより学習者は導入直後から過去の間違いを復習できる。同一 (市区町村, 出題モード) が複数回誤答されていても復習対象は1件に集約する。
- **FR-002**: 復習対象の単位は (市区町村コード, 出題モード) の組み合わせとし、同一市区町村でもモードが異なれば独立した復習状態として管理しなければならない。
- **FR-003**: システムは、各復習対象について「次回復習期日」「連続正解回数」「現在の復習間隔」「個別の難易度係数（容易さ）」「最終復習日時」を保持しなければならない。

**間隔反復による再スケジュール（アクティブリコール）**

- **FR-004**: 復習は学習者が自力で答えを想起して回答する形式（アクティブリコール）でなければならず、回答する前に正解を提示してはならない。
- **FR-005**: システムは、回答結果から「回答の質」を判定し、確立された間隔反復方式（SM-2、Assumptions 参照）に従って各復習対象の次回復習期日・間隔・連続正解回数・難易度係数を更新しなければならない。
- **FR-005a**: SM-2 の更新は、復習セッションの回答に限らず、通常クイズで復習登録済みの (市区町村, 出題モード) に回答した場合にも適用しなければならない（正解で前進＝間隔拡大・卒業、誤答でリセット＝連続正解0・短間隔）。同一復習対象に同日複数回回答した場合、正解による前進は **JST 1日1回まで**（同日2回目以降の正解はログのみで前進しない）。誤答によるリセット/復帰は同日でも常に適用する。
- **FR-006**: 正解（質が合格基準以上）の場合、次回復習間隔は前回より広がり、連続正解が続くほど間隔が長くならなければならない。
- **FR-007**: 誤答（質が合格基準未満）の場合、連続正解回数をリセットし、次回復習期日を短い間隔（翌日相当）に戻さなければならない。
- **FR-008**: システムは、新規登録時に初回復習期日を設定しなければならない。新規の誤答による登録は**翌日相当**を初回期日とする（FR-007 と整合）。導入時バックフィル分（FR-001a）は例外的に**即時到来**とする。

**復習セッションの出題**

- **FR-009**: 復習セッションは、次回復習期日が現在時刻以前（期日到来済み）の復習対象のみを出題対象としなければならない。期日が未到来の復習対象は、学習者が望んでも出題してはならない（スケジュール無視のオンデマンド復習・先取り復習は提供しない）。
- **FR-010**: 期日到来の復習対象が1回のセッション出題上限を超える場合、優先順位（期日が古い順、次いで定着度が低い順）に従って上限件数まで出題し、残りは持ち越さなければならない。
- **FR-011**: 同一の復習対象を、同一セッション内で重複して出題してはならない。
- **FR-011a**: 復習セッションでの各回答は、通常クイズと同様にクイズ回答ログ（正誤・モード・回答日時）へ記録しなければならない。これにより復習回答も既存のダッシュボード指標（正答率推移・苦手ランキング等）へ反映される。
- **FR-012**: 復習セッションで出題する設問形式は、その復習対象が属する出題モードで提示しなければならない。
- **FR-012a**: 1回の復習セッションは複数の出題モードが混在しうる（例: 期日到来10件が Mode A 2件・B 3件・C 3件・D 2件）。システムは、問題ごとに該当モードの設問UI（地図選択・4択・地図タップ等）へ切り替えながら、1つの連続したセッションとして出題しなければならない。
- **FR-012b**: 混在セッションの出題順は、優先順位（期日が古い順、次いで定着度が低い順）に従い、モードでまとめ直さずに混在のまま出題する（交互練習＝インターリービングを既定とする）。

**ダッシュボード表示（アプリ内のみ）**

- **FR-013**: システムは、ダッシュボードの復習導線に「今日復習すべき件数（期日到来済み件数）」を表示しなければならない。
- **FR-014**: 学習者は、ダッシュボードの復習導線から、期日到来分のみで構成された復習セッションを開始できなければならない。
- **FR-015**: 期日到来済みの復習対象が0件の場合、システムは「今日の復習なし」を明示し、次回復習予定（最短の次回期日）を案内しなければならない。
- **FR-016**: システムは、復習の進捗（復習中の件数、定着済みの件数、今後の復習予定）を学習者が確認できるよう表示しなければならない。
- **FR-016a**: システムは、「覚えている途中（復習中）の市区町村一覧」を、メタ認知・進捗可視化として表示してよい。各項目は市区町村名・出題モード・次回復習日を示すが、**答え（都道府県等）は表示してはならない**（受動的な答えの再露出による流暢性の錯覚を避けるため）。この一覧は情報表示であり、出題（アクティブリコール）の代替ではない。
- **FR-016b**: 件数が多くなりうるため、ダッシュボードには一覧そのものを埋め込まず「復習中の件数＋一覧への導線」のみを置き、全件は専用ページで表示する。専用ページは**ページング**と**モード別フィルタ**を備える。
- **FR-016c**: 専用ページ先頭に、状況を一目で把握できる**モード別サマリ表**（A/B/C/D × 復習中件数・定着済み件数・定着率）を表示する。
- **FR-017**: 本機能はアプリ内表示のみとし、メール・プッシュ通知等のアプリ外リマインドは含まない。

**定着（マスター）の扱い**

- **FR-018**: システムは、十分に長い間隔かつ十分な連続正解に達した復習対象を「定着済み（卒業）」とし、以後は復習スケジュールから外して出題対象・「今日の復習」件数のいずれからも除外しなければならない。
- **FR-019**: 定着済みの復習対象が後で（通常クイズ等で）誤答された場合、再び復習対象（連続正解リセット・短間隔・期日即時到来）に復帰させなければならない。

### Key Entities *(include if feature involves data)*

- **復習対象 (Review Item)**: ある学習者の、特定の (市区町村, 出題モード) に対する間隔反復の学習状態。主な属性: 学習者・市区町村コード・市区町村名・都道府県・出題モード・次回復習期日・現在の復習間隔（日数）・連続正解回数・難易度係数（容易さ）・最終復習日時・状態（復習中／定着済み）。1人の学習者は多数の復習対象を持つ。
- **クイズ回答ログ (Quiz Result)**: 既存の、学習者の各回答記録（正誤・モード・回答日時）。間違いの捕捉と「回答の質」判定の入力源。本機能の状態管理（復習対象）とは別レイヤーで、ログは引き続き蓄積される。復習セッションでの回答もこのログに記録される（FR-011a）。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 学習者は、ダッシュボードを開いてから3タップ以内で、期日到来分のみの復習セッションを開始できる。
- **SC-002**: 誤答した問題は、回答確定の直後（体感遅延なく）に復習対象として登録/更新される。新規の誤答は翌日相当が初回期日となるため当日の「今日の復習」件数には入らず、翌日以降に数えられる。導入時にバックフィルされた過去の誤答（FR-001a）は即時到来で当日の件数に反映される。
- **SC-003**: 復習で正解を重ねるほど次回出題までの間隔が単調に長くなり、誤答すると翌日相当へ短縮されることを、復習対象の期日推移として検証できる。
- **SC-004**: 同一市区町村でも出題モードが異なる復習対象は独立して期日管理され、片方の復習が他方の期日に影響しない。
- **SC-005**: ある間違いを間隔反復に沿って繰り返し復習した学習者は、その市区町村×モードの再出題で正答率が向上する（導入前の単純な「誤答×7日」方式と比較して、同一問題の再正答までの復習回数の無駄が減る）。
- **SC-006**: 期日到来分が1セッション上限を超える場合でも、出題件数は常に上限以下に保たれ、優先度の高い（期日が古い／定着度が低い）問題から出題される。
- **SC-007**: 複数モードが混在する期日到来分（例: A2/B3/C3/D2）を、学習者は1つの復習セッションとして中断なく回答でき、各問題が登録モードの設問形式で提示される。
- **SC-008**: 期日が未到来の復習対象は、復習セッションにも「今日の復習」件数にも一切現れない。

## Assumptions

- **間隔反復アルゴリズム**: SM-2（SuperMemo-2）を採用する。各復習対象は難易度係数（ease factor, 初期値 2.5・下限 1.3）、連続正解回数（repetition）、間隔（interval, 日数）を持ち、回答の質（0〜5）に基づいて次回間隔を算出する。質が3未満なら連続正解回数を0に戻し間隔を初期（1日相当）へ、3以上なら repetition を進めて間隔を拡大する。これにより固定の Leitner 箱ではなく1問ごとに最適化された間隔となる。
- **回答の質マッピング（二値→SM-2 品質）**: クイズの「正解／不正解」の二値を SM-2 の質へマッピングする。不正解＝2、正解＝4 で固定し、回答時間やその他要因による補正は行わない（ヒント機能は存在しない）。質4は連続正解で間隔を拡大し、質2未満扱いの不正解は連続正解をリセットして間隔を初期に戻す（FR-006/007）。回答時間に基づく多段階化は将来拡張の余地として残すがスコープ外。
- **管理単位**: (市区町村コード, 出題モード) 単位。Mode A は同名市区町村を1問にまとめる出題仕様のため、復習対象の市区町村キーは既存のコード単位（または Mode A の出題キー）に合わせて実装時に統一する。
- **既存誤答履歴の扱い**: 本機能導入時に既存の誤答ログから復習対象をバックフィルし、初回期日は即時到来とする（FR-001a）。大量バックフィル時の負荷分散（一気に到来させず分散配置するか）や、バックフィルの実行タイミング（マイグレーション時の一括 or 初回ログイン時の遅延生成）は plan で確定する。
- **入口**: 既存ダッシュボードの「復習おすすめ」付近を本機能の入口として置き換え／統合する。既存の「誤答×7日経過×誤答率順」ロジックは本機能の期日ベース選定に置き換わる。
- **復習時のモード**: 復習はその復習対象が登録された出題モードで出題する（学習者による任意モード変更は本 spec のスコープ外）。1セッションはモード混在で、問題ごとに設問UIを切り替えて出題する。出題順は優先度順（期日が古い／定着度が低い順）で、モードはまとめ直さずインターリービングする。
- **期日駆動のみ（出題対象）**: 復習セッションで出題するのは期日が到来したものだけ。期日未到来の問題を先取りで復習する導線や、スケジュールを無視した自由ドリルは提供しない（間隔反復の科学的妥当性を保つため）。ただし通常クイズで偶然出題され回答した場合は、期日前でも SM-2 を更新する（FR-005a）。
- **同日重複回答の扱い**: 同一復習対象に同日複数回回答した場合、連続して間隔が過度に伸びないよう反映上限を設ける想定（例: 1日1回まで前進）。具体ルールは plan で確定する。
- **セッション出題上限**: 既存の通常クイズと同等の1セッション問題数上限を踏襲する（具体値は plan で確定）。
- **「定着済み（卒業）」の判定基準**: 間隔が一定日数以上（例: 30日相当以上）かつ連続正解が一定回数以上に達した状態を定着済み＝卒業とみなし、復習スケジュールから外す（具体閾値は plan で確定）。卒業後に通常クイズ等で誤答すると復習対象へ復帰する（FR-019）。
- **既存資産の再利用**: 既存のクイズ回答ログ・出題エンジン・難易度／地域の概念・苦手優先サンプリング等の仕組みを前提に、復習対象の状態管理レイヤーを追加する形で実装される想定。
- **認証**: 既存のユーザー認証・ユーザー単位のデータ分離（行レベルの保護）をそのまま利用する。
