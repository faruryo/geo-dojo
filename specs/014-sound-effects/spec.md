## User Scenarios & Testing *(mandatory)*

### User Story 1 - 正解・不正解を音で即座に感じ取る (Priority: P1)

クイズをプレイ中のユーザーは、1問解答するたびに、その解答が正解か不正解かを、既存の視覚フィードバック（✓/✗ 表示・色分け）に加えて効果音でも即座に感じ取れる。正解音と不正解音は明確に聞き分けられる。

**Why this priority**: 現状のフィードバックは色とテキストのみで、テンポよく連続解答するクイズにおいて手応えが薄い。1問ごとの音のフィードバックこそが「いい感じのSE」の中核体験であり、これがなければ本機能の価値は成立しないため最優先とする。

**Independent Test**: いずれかのクイズ（市区町村クイズ・都道府県クイズ・復習セッション）で1問解答し、正解時に正解を表す効果音が、不正解時にはそれと聞き分けられる別の効果音が鳴ることを確認する。完了時演出音（P2）やミュート機能（P3）が未実装でも、これ単体で「解答の手応えが増す」価値が成立する。

**Acceptance Scenarios**:

1. **Given** クイズの問題が表示されている、**When** ユーザーが解答して正解する、**Then** 正解を表す効果音が再生される。
2. **Given** クイズの問題が表示されている、**When** ユーザーが解答して不正解になる、**Then** 正解時とは聞き分けられる、不正解を表す効果音が再生される。
3. **Given** 制限時間のある出題形式（地図タップ形式）で解答しないまま時間が切れた、**When** タイムアウトが不正解として処理される、**Then** 不正解時と同じ効果音が再生される。
4. **Given** 効果音が再生される、**When** フィードバック表示から次の問題への自動遷移が行われる、**Then** 効果音の再生を待つことによる遷移の遅延は発生しない（既存のテンポが維持される）。

---

### User Story 2 - クイズ完了時に達成感のある音を聞く (Priority: P2)

1回分のクイズを最後まで解き終えたユーザーは、結果画面が表示されるタイミングで完了を告げる効果音を聞く。全問正解で終えた場合は、通常の完了時とは異なる、より祝祭感のある演出音が鳴る。

**Why this priority**: 1問ごとのフィードバック（P1）がクイズ体験の基礎であるのに対し、完了音はプレイ全体を締めくくり達成感を高める上乗せの演出。P1と独立に実装・検証できるが、鳴る頻度が低く（1プレイに1回）体験への寄与も相対的に小さいため次点とする。

**Independent Test**: クイズを最後まで解き終えて結果画面に到達し、完了を告げる効果音が再生されることを確認する。全問正解で終えた場合に、それ以外の成績で終えた場合と聞き分けられる演出音が鳴ることを確認する。

**Acceptance Scenarios**:

1. **Given** クイズの最後の1問に解答し終えた、**When** 結果画面が表示される、**Then** クイズ完了を告げる効果音が再生される。
2. **Given** 全問正解でクイズを完了した、**When** 結果画面が表示される、**Then** 通常の完了音とは聞き分けられる、より際立った演出音が再生される。
3. **Given** クイズをプレイ中である、**When** 「中断」操作でクイズを途中終了する（結果画面に至らない）、**Then** 完了時の効果音は再生されない。
4. **Given** 復習セッションで「続けて復習する」により連続プレイしている、**When** 各バッチの結果画面が表示される、**Then** バッチごとに完了音が再生される（連続プレイ中も1バッチ完了ごとに鳴る）。

---

### User Story 3 - 音を消したいときにミュートする (Priority: P3)

電車内や職場など音を出したくない環境のユーザーは、クイズ画面上のワンタップ操作で効果音をミュートでき、以後すべての効果音が鳴らなくなる。ミュート状態は同じ端末での次回訪問時も維持される。

**Why this priority**: 効果音そのもの（P1・P2）がなければミュートは意味を持たないため優先度としては最後。ただし、モバイルPWAとして公共の場での利用が主想定であり、端末側のマナーモード等に頼らずアプリ内で明示的に消せる手段は、効果音リリースと同時に提供すべき必須の付随機能である。

**Independent Test**: クイズ画面でミュート操作を行い、以後の解答・完了で効果音が一切鳴らないこと、視覚フィードバックは通常どおり機能し続けることを確認する。ブラウザを閉じて再訪問し、ミュート状態が維持されていることを確認する。再度の操作でミュートが解除でき、音が復帰することを確認する。

**Acceptance Scenarios**:

1. **Given** クイズ画面が表示されている、**When** ユーザーがミュート操作を行う、**Then** 以後の正解・不正解・完了のすべての効果音が再生されなくなる。
2. **Given** ミュート状態でプレイしている、**When** 解答する、**Then** 効果音は鳴らないが、正解/不正解の視覚フィードバック（✓/✗・色分け）はミュート前と変わらず表示される。
3. **Given** ミュートに設定してアプリを離れた、**When** 同じ端末・同じブラウザで改めてアプリを開いてプレイする、**Then** ミュート状態が引き継がれており、音は鳴らない。
4. **Given** ミュート状態である、**When** ユーザーが再度ミュート操作を行う（解除）、**Then** 以後の解答・完了時に効果音が再び再生される。
5. **Given** 市区町村クイズでミュートした、**When** 都道府県クイズや復習セッションをプレイする、**Then** そちらでも効果音は鳴らない（設定はクイズ画面横断で共通）。

---

### Edge Cases

- ブラウザの自動再生制限により、ユーザー操作を経ていない状態では音声を再生できない場合がある → クイズの解答は常にユーザーのタップ/クリックを起点とするため通常フローでは問題にならないが、タイムアウトによる不正解処理など操作を伴わない発火で再生が拒否された場合でも、正誤判定・進行・視覚フィードバックはブロックされてはならない（音だけが黙って省略される）。
- 端末がサイレントモード・マナーモードである場合 → OS側の制御で音が聞こえないのは正常な挙動であり、アプリ側で特別な検知・警告は行わない。アプリ内でミュートされていない限り再生は試みてよい。
- 音源の読み込み・初期化・再生が失敗した場合 → 当該回の効果音のみを諦め、クイズの判定・進行・結果表示には一切影響を与えない。エラーがユーザーに見える形で表面化してはならない。
- 短時間に連続して解答が行われ、前の効果音の再生完了前に次の効果音が発火する場合 → 音が重なる・前の音が打ち切られる等はどちらでも許容するが、クイズの進行速度や判定に影響してはならない。
- PWAとしてオフラインで利用している場合 → オフラインでプレイ可能な画面では、効果音もネットワークなしで再生できなければならない（音のためだけに通信を必要としない）。
- タブがバックグラウンド・非アクティブな状態での再生可否 → ブラウザの標準挙動に委ね、アプリ側で特別な制御は行わない。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは、クイズの1問に正解した際、正解を表す効果音を再生しなければならない。
- **FR-002**: システムは、クイズの1問に不正解となった際（制限時間切れによる不正解処理を含む）、正解音とは明確に聞き分けられる不正解を表す効果音を再生しなければならない。
- **FR-003**: システムは、1回分のクイズを最後まで解き終え結果画面が表示される際、クイズ完了を告げる効果音を再生しなければならない。復習の連続プレイでは1バッチの完了ごとに再生する。
- **FR-004**: システムは、全問正解でクイズを完了した場合、通常の完了音とは聞き分けられる、より際立った演出効果音を再生しなければならない。
- **FR-005**: システムは、クイズを中断して結果画面に至らなかった場合、完了時の効果音（全問正解演出を含む）を再生してはならない。
- **FR-006**: 効果音は、正誤判定と完了状態を持つすべてのクイズ画面（市区町村クイズのモードA〜D、都道府県クイズ、復習セッション）で一貫して提供されなければならない。同じ意味のイベント（正解・不正解・完了）には画面をまたいで同じ音を用いる。
- **FR-007**: 各効果音は短く控えめなものとし、効果音の再生がフィードバック表示時間や次の問題への遷移タイミングを遅らせてはならない（既存のプレイテンポを変えない）。
- **FR-008**: ユーザーは、クイズ画面上の操作で効果音の再生有無を切り替え（ミュート/ミュート解除）できなければならない。
- **FR-009**: ミュート設定は、対象となるすべてのクイズ画面に共通して適用される単一の設定でなければならない（画面ごとに別々の設定を持たない）。
- **FR-010**: ミュート設定は、同一端末・同一ブラウザにおける次回以降の訪問時にも維持されなければならない。
- **FR-011**: ミュート中であっても、正解・不正解の視覚フィードバック（✓/✗表示・色分け）およびクイズの進行は、効果音の有無に関わらず通常どおり機能しなければならない。
- **FR-012**: 効果音の読み込み・再生に失敗した場合、または再生がブラウザに拒否された場合でも、システムはクイズの正誤判定・問題の進行・結果画面への遷移をブロックしてはならず、エラーをユーザーに表面化させてはならない。
- **FR-013**: 使用する音源は、本アプリでの配布・再生についてライセンス上の問題がないもの（自作・生成音またはそれに準ずる利用許諾を持つもの）でなければならない。
- **FR-014**: オフラインでプレイ可能な画面では、効果音もネットワーク接続なしで再生できなければならない。

### Key Entities *(include if feature involves data)*

- **効果音イベント**: 効果音の再生トリガーとなる出来事の分類。「正解」「不正解」「クイズ完了（通常）」「クイズ完了（全問正解）」の4種類。どのタイミングでどの種類の音が鳴るべきかを表す概念であり、具体的な音源そのものではない。
- **ミュート設定**: 効果音の再生を許可するかどうかを表す、端末（ブラウザ）単位で保持される二値の状態。すべてのクイズ画面に横断して適用され、次回訪問時にも引き継がれる。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 対象となるすべてのクイズ画面（市区町村クイズ全モード・都道府県クイズ・復習セッション）で、解答のたびに正解と不正解を聞き分けられる効果音が鳴る。
- **SC-002**: クイズを完了するたびに完了音が鳴り、全問正解時はそれ以外の完了と聞き分けられる演出音が鳴る。中断時には鳴らない。
- **SC-003**: ユーザーはワンタップの操作でミュートを切り替えられ、切り替えた状態は同一端末での次回訪問時にも維持されている。
- **SC-004**: ミュート状態では、いかなる効果音（正解・不正解・完了・全問正解演出）も再生されない。
- **SC-005**: 効果音の再生失敗・再生拒否・ミュートのいずれの状態でも、クイズの完走（最後の問題まで到達し結果画面が表示されること）と1問ごとの遷移テンポは、効果音導入前と変わらず保たれる。

## Assumptions

- **対象範囲**: 効果音の対象は、正誤判定と完了状態を持つ既存のすべてのクイズ画面（市区町村クイズのモードA〜D、都道府県クイズ、復習セッション）とする。都道府県クイズは市区町村クイズと別実装だが、ユーザーから見れば同じ「クイズ」であり片方だけ音が鳴るのは一貫性を欠くため、両方を本featureの範囲に含める。今後追加される新しいクイズ形式への適用は将来判断とする。
- **音量調整のスコープ**: backlog の「音量調節」については、本featureでは段階的な音量スライダーは設けず、ミュート/ミュート解除の二値切り替えのみを提供する。細かな音量調整は端末の音量ボタンで代替可能であり、段階調整UIが必要になった場合は将来のbacklogとして扱う。
- **設定UIの置き場所**: 現状このリポジトリには汎用の設定画面が存在しない。本featureのために専用の設定画面を新設するのは過剰なため、ミュート切り替えはクイズ画面内に配置する小さな操作（アイコントグル等）で提供する。将来設定画面が作られた際にそこへ統合することは妨げない。
- **永続化の範囲**: ミュート設定の維持は「同一端末・同一ブラウザでの次回訪問時」までを要件とし、アカウントに紐づく複数端末間の同期（サーバー側保存）はスコープ外とする。効果音の好みは端末の利用環境（自宅か外出先か）に依存するため、端末ローカルの保持で十分と判断する。
- **初期状態**: ミュート設定が未保存の初回利用時のデフォルトは「音あり」とする。本機能の価値を体験してもらうことを優先し、不要なユーザーは明示的にミュートする。
- **全問正解の判定基準**: 全問正解演出は1回分のクイズ（復習では1バッチ）内の全問正解時にのみ発火する。それ未満の成績（9割正解など）はすべて通常の完了音とし、段階的な成績別演出はスコープ外とする。
- **音の性格**: 効果音は「短い・控えめ・不快でない」ことを要件とし（FR-007）、具体的な音色・長さ・音量バランスの選定は実装時のデザイン判断に委ねる。BGM（継続的な背景音楽）は本featureのスコープ外とする。
- **実現方式**: 効果音の生成・再生方式（合成音の動的生成か音声アセットファイルか、使用API等）は実装判断であり、本仕様書は「いつ・どう聞こえるべきか」というふるまいと、ライセンス安全性（FR-013）・オフライン再生可能性（FR-014)という制約のみを定義する。
- **既存挙動への影響**: 本featureは効果音という付加的フィードバックの追加のみを行い、正誤判定ロジック・フィードバック表示時間・結果集計・間隔反復のスケジューリングなど既存の挙動には一切変更を加えない。
