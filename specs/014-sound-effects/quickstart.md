# Quickstart: クイズ効果音（SE）の追加

## テスト実行

```bash
pnpm test     # Vitest（sound-effects のユニットテスト含む）
pnpm lint     # 型チェック / Lint
```

## 動作確認（ローカル）

### 準備

```bash
supabase start   # ローカル Supabase（起動していない場合のみ）
pnpm dev
```

ブラウザで `http://127.0.0.1:3000` を開く。**端末・PC の音量が出る状態にしておく**（マナーモード解除、音量アップ）。

> [!NOTE]
> **ログインについて**
> ローカル環境ではメール確認が無効化されています。テストユーザー `test@example.com` / `password123`、または `/signup` から任意のメール＋パスワード（6文字以上）で即ログインできます。

### 動作確認手順

#### 1. 正解・不正解の効果音（US1 / FR-001, FR-002）

1. `/quiz/municipality` からモードB（逆引き4択）を開始する。
2. 正しい選択肢をタップし、✓ 表示と同時に正解音が鳴ることを確認する。
3. わざと誤答し、✗ 表示と同時に**正解音とは明確に聞き分けられる**不正解音が鳴ることを確認する。
4. モードA（地図タップ）・モードC（順引き4択）・モードD（順引き地図）でも同様に正解音・不正解音が鳴ることを確認する。
5. 復習セッション（`/quiz/review`。復習期日の項目が必要）と都道府県クイズ（`/quiz/prefecture`）でも、**同じ**正解音・不正解音が鳴ることを確認する（FR-006: 画面をまたいで同じ音）。

#### 2. タイムアウト時の不正解音（US1 シナリオ3）

1. モードDを開始し、解答せずに30秒待つ。
2. タイムアウトで不正解処理された瞬間に、不正解音（手動誤答時と同じ音）が鳴ることを確認する。
3. 音の有無に関わらず正解表示・次の問題への遷移が通常どおり行われることを確認する。

#### 3. プレイテンポの非破壊（US1 シナリオ4 / FR-007 / SC-005）

1. モードB で10問をテンポよく連続解答する。
2. 効果音導入前と同じ体感で次の問題へ遷移すること（フィードバック表示が長くなっていない・音の再生完了を待っていない）を確認する。
3. 前の音が鳴り終わる前に次の解答をしても、進行が詰まらないことを確認する（spec Edge Cases）。

#### 4. 完了音・全問正解演出（US2 / FR-003, FR-004, FR-005）

1. モードB を10問で開始し、わざと1問以上誤答して完走する。結果画面表示と同時に**通常の完了音**が鳴ることを確認する。
2. 再度プレイして全問正解する。結果画面表示と同時に、通常の完了音とは聞き分けられる**より際立った演出音**が鳴る（通常完了音と重ねて鳴らない）ことを確認する。
3. プレイ途中で「中断」をタップする。完了音・演出音が**鳴らない**ことを確認する。
4. 都道府県クイズでも 1〜3 を同様に確認する（10問固定）。
5. 復習セッションで完走→「続けて復習する」で次バッチも完走し、**バッチごとに**完了音が鳴ることを確認する（US2 シナリオ4）。

#### 5. ミュートトグル（US3 / FR-008〜FR-011）

1. クイズプレイ中のヘッダにスピーカーアイコンがあることを確認し、タップしてミュートにする（アイコンがミュート表示に変わる）。
2. 以後の解答・完走で、正解・不正解・完了・全問正解の**いずれの音も鳴らない**こと（SC-004）、✓/✗ の視覚フィードバックと進行は通常どおりであること（FR-011）を確認する。
3. 市区町村クイズでミュートした状態のまま、都道府県クイズ・復習セッションを開き、そちらでも音が鳴らないことを確認する（FR-009: 画面横断の単一設定）。
4. 再度アイコンをタップしてミュート解除し、次の解答から音が復帰することを確認する。

#### 6. ミュート状態の永続化（US3 シナリオ3 / FR-010 / SC-003）

1. ミュートに設定した状態でタブ／ブラウザを完全に閉じる。
2. 同じブラウザで再度アプリを開いてプレイし、ミュート状態が引き継がれている（音が鳴らず、アイコンもミュート表示）ことを確認する。
3. DevTools → Application → Local Storage で `geo-dojo:se-muted` = `true` を確認する。キーを削除してリロードすると「音あり」（デフォルト）に戻ることを確認する。

#### 7. 再生失敗時のフォールバック（Edge Cases / FR-012）

1. DevTools の Console で `AudioContext` を一時的に壊す（例: `window.AudioContext = function(){ throw new Error('boom') }` を実行）か、`playSe` 内に一時的な `throw` を仕込む。
2. その状態で解答・完走しても、エラー表示・進行停止が発生せず、音だけが省略されることを確認する（Console に error レベルの出力が出ないこと）。

#### 8. オフライン再生（FR-014）

1. `pnpm build && pnpm start` で本番ビルドを起動し（serwist はビルド時のみ有効）、一度アプリを開いて PWA キャッシュを作る。
2. DevTools → Network → Offline に切り替え、オフラインでプレイ可能な画面（都道府県クイズ等）を開く。
3. 解答時に効果音が通常どおり鳴ることを確認する（音のための追加ネットワークリクエストが発生しないこと。Web Audio 生成音のため、Network タブに音声ファイルの取得が現れないことも確認できる）。

#### 9. 端末のマナーモード（Edge Cases・参考確認）

1. iOS/Android 実機でマナーモードにしてプレイし、アプリが警告等を出さないこと（OS の制御に委ねる）を確認する。アプリ内ミュートでない限り再生は試みられる。
