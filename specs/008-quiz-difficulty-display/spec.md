# Feature Specification: 市区町村クイズの問題に難易度を表示

**Feature Branch**: `008-quiz-difficulty-display`  
**Created**: 2026-06-16  
**Status**: Draft  
**Input**: User description: "市区町村クイズで問題を表示する時に、その問題がどの難易度なのかを表示する"

## Clarifications

### Session 2026-06-16

- Q: モードAで同名の市区町村が複数都道府県に存在し難易度が異なる場合、表示する難易度をどのルールで決めるか → A: 出題対象のうち最も高い（難しい）難易度を代表値として表示する

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 出題中に難易度がわかる (Priority: P1)

市区町村クイズをプレイ中のユーザーが、いま表示されている問題が4段階（入門・中級・上級・達人）のどの難易度に属する市区町村なのかを、問題ごとに一目で把握できる。

**Why this priority**: これが本機能の中核。これ単体で「出題中に難易度が見える」という価値が完結し、リリース可能な最小単位になる。出題ごとに自分が今どのレベルに挑戦しているのかが分かると、正誤の納得感と学習のメリハリが生まれる。

**Independent Test**: 任意のモード（A/B/C/D）で市区町村クイズを開始し、各問題カードに難易度ラベルが表示されること、次の問題に進むとその問題の難易度に更新されることを確認すれば、単独で検証・価値提供できる。

**Acceptance Scenarios**:

1. **Given** ユーザーが市区町村クイズを開始し問題が表示されている、 **When** 問題カードを見る、 **Then** その問題の市区町村に対応する難易度（入門／中級／上級／達人のいずれか）がラベルとして表示される
2. **Given** ある問題に解答して次の問題に進んだ、 **When** 次の問題が表示される、 **Then** 難易度ラベルは新しい問題の難易度に更新される
3. **Given** 複数の難易度を混在させて出題する設定（おまかせ／推薦など）でプレイしている、 **When** 難易度の異なる問題を連続して解く、 **Then** 各問題でその問題自身の難易度が正しく表示される

---

### User Story 2 - すべての出題モードで一貫して見える (Priority: P2)

ユーザーが地図タップ（モードA・D）、選択肢（モードB・C）、および復習クイズのいずれをプレイしていても、同じ位置・同じ表現で難易度ラベルを確認できる。

**Why this priority**: P1 を全モードへ広げて体験を統一する。モードごとに表示有無や見え方がバラつくと混乱を生むため、一貫性が重要だが、まず1モードで価値が出せるため P2。

**Independent Test**: モードA・B・C・D および復習クイズをそれぞれ開始し、いずれの問題カードにも難易度ラベルが同じ要領で表示されることを確認する。

**Acceptance Scenarios**:

1. **Given** モードA（都道府県を地図でタップ）で出題されている、 **When** 問題カードを見る、 **Then** 難易度ラベルが表示される
2. **Given** モードC（都道府県名から市区町村を選ぶ）で出題されている、 **When** 問題カードを見る、 **Then** 正解となる市区町村の難易度ラベルが表示される
3. **Given** 復習クイズで出題されている、 **When** 問題カードを見る、 **Then** 通常クイズと同じ要領で難易度ラベルが表示される

---

### Edge Cases

- **難易度データが欠落している市区町村**: マスタに難易度が設定されていない問題が出題された場合、ラベル領域を非表示にする（誤った難易度を表示しない／レイアウトを崩さない）。
- **モードAで同名の市区町村が複数の都道府県に存在**: 出題対象の市区町村群の難易度が一致しないケースがあり得る。この場合、出題対象のうち最も高い（難しい）難易度を代表値として表示する（FR-007）。
- **正誤フィードバック表示中**: 解答直後のフィードバック表示中も難易度ラベルは引き続き表示され、消えたり切り替わったりしない。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは、市区町村クイズの各問題表示時に、その問題に対応する難易度を画面上に表示しなければならない。
- **FR-002**: 難易度の表示は既存の4段階区分（入門・中級・上級・達人）と既存の表記（星付きラベル）に一致しなければならない。
- **FR-003**: 問題が切り替わるたびに、難易度表示はその時点の問題の難易度へ更新されなければならない。
- **FR-004**: 難易度表示は全出題モード（A・B・C・D）および復習クイズで一貫して提供されなければならない。
- **FR-005**: 出題対象の難易度が判定できない場合、システムは誤った難易度を表示せず、難易度表示を省略しなければならない。
- **FR-006**: 難易度表示は問題文・解答操作の視認性を妨げず、正解のヒントとして機能する情報（都道府県名・市区町村名など、その問題で問われている答え）を新たに露出してはならない。
- **FR-007**: モードAのように複数の市区町村が出題対象となる場合、表示する難易度は出題対象のうち最も高い（難しい）難易度を代表値として決定しなければならない。

### Key Entities *(include if feature involves data)*

- **問題（Question）**: ユーザーに提示される1問。対象となる市区町村（1件または複数件）と出題モードを持つ。本機能では、この問題に紐づく難易度を表示対象とする。
- **難易度（Difficulty）**: 市区町村マスタが各市区町村に対して持つ4段階の区分。入門・中級・上級・達人。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 市区町村クイズの全出題モードおよび復習クイズで、出題されたすべての問題（難易度データを持つもの）に対し難易度ラベルが表示される（表示率100%）。
- **SC-002**: ユーザーは問題カードを見た時点で、追加の操作なしにその問題の難易度を把握できる。
- **SC-003**: 難易度表示の追加によって、既存の出題・解答・採点・進行（次の問題への遷移、結果集計）の動作に影響が出ない。
- **SC-004**: 難易度ラベルの追加でレイアウト崩れや問題文・選択肢・地図の視認性低下が発生しない。

## Assumptions

- 各市区町村の難易度は既存の市区町村マスタに既に存在しており、本機能は新たな難易度の算出・採番は行わない（表示のみ）。
- 難易度は出題対象の市区町村そのものの属性として扱い、ユーザーの習熟度や正答率に応じて動的に変化させるものではない。
- 難易度の段階数・名称・表記（入門／中級／上級／達人の星付きラベル）は現行のものを踏襲し、本機能で再定義しない。
- 表示位置は各問題カード内（問題文の近傍）を想定し、既存の進捗表示（「N / M」「N 正解」「残り秒数」）と競合しない範囲に配置する。
- モードCでは問われている答えは市区町村側だが、難易度は市区町村の属性であるため表示してもヒントの過剰露出には当たらない（FR-006 の趣旨に反しない）ものとする。
