# Feature Specification: 誤答なし市区町村の早期卒業

**Feature Branch**: `009-fast-graduation`
**Created**: 2026-07-06
**Status**: Draft
**Input**: User description: "一度も間違えたことがない市区町村をSRSから早期卒業させる。最初から知っている市区町村がいつまでも「覚えている途中」に残り続ける問題を解消する。"

## 背景 / 問題

現行の復習スケジュール（005-spaced-review の SM-2）では、卒業条件が「復習間隔 30日以上 かつ 正解 4回以上」の一律コースしかない。このため、ユーザーが最初から知っている市区町村（一度も間違えたことがないもの）でも、最短で「4回正解 × 約22日間」かけないと卒業できず、「覚えている途中の市区町村」一覧がいつまでも減らない。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 誤答なしなら2回連続正解で即卒業 (Priority: P1)

最初から知っている市区町村は、一度も間違えずに別日で2回正解した時点で「覚えた（卒業）」扱いになり、「覚えている途中」一覧から消える。まぐれ正解（地図タップでたまたま近くを押した等）は2回目の確認でふるい落とされる。

**Why this priority**: 本機能の核。ユーザーの主訴（簡単な問題がいつまでも終わらない）を直接解消する。

**Independent Test**: 新規の市区町村に別日で2回正解し、卒業済みになることを確認できる。

**Acceptance Scenarios**:

1. **Given** 一度も出題されたことがない市区町村, **When** 正解し、翌日以降にもう一度正解する, **Then** その時点で卒業済みになり「覚えている途中」一覧に表示されない
2. **Given** 過去に誤答したことがある市区町村, **When** 2回連続で正解する, **Then** 卒業せず、従来どおりのコース（間隔30日以上かつ正解4回以上）が適用される
3. **Given** 一度も誤答がなく正解1回の市区町村, **When** 同じ日にもう一度正解する, **Then** 同日の正解は前進としてカウントされず（既存の同日ガード）、卒業しない
4. **Given** 早期卒業した市区町村, **When** その後の出題で誤答する, **Then** 従来どおり「覚えている途中」に復帰し、以後は誤答ありのため通常コースになる

---

### User Story 2 - 既存レコードの一括卒業 (Priority: P2)

すでに「誤答なしで2回以上正解済み」の状態にある既存の市区町村は、機能導入時に一括で卒業済みへ移行され、「覚えている途中」一覧が実態に合った件数まで一気に減る。

**Why this priority**: これがないと既存の溜まったレコードは新ルールの恩恵を受けるのに再度全件解き直す必要があり、主訴が解消されない。

**Independent Test**: 移行処理の前後で「覚えている途中」件数を比較し、条件該当分だけ減ることを確認できる。

**Acceptance Scenarios**:

1. **Given** 誤答履歴がなく正解2回以上の既存レコード, **When** 一括移行を実行する, **Then** 卒業済みになる
2. **Given** 誤答履歴がある、または正解1回以下の既存レコード, **When** 一括移行を実行する, **Then** 状態は変わらない
3. **Given** 一括移行を一度実行済み, **When** 再度実行する, **Then** 結果は変わらない（冪等）

---

### Edge Cases

- 卒業直前に誤答した場合: 誤答時点で通常のリセット処理が走り、以後は誤答ありとして扱われる
- 同一市区町村でもクイズモードが異なる場合: 学習状態はモードごとに独立しているため、誤答の有無・卒業判定もモード単位で行う
- 誤答履歴の判定は本人の全期間の回答履歴に基づく（復習機能導入前の回答も含む）
- 一括移行は本番データに対して1回実行するが、再実行しても安全（冪等）であること

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: System MUST 正解時の卒業判定に「その市区町村×モードに誤答履歴が一切なく、かつ別日での正解が2回に達した場合は即卒業」の条件を追加する
- **FR-002**: System MUST 既存の卒業条件（復習間隔30日以上 かつ 正解4回以上）を従来どおり維持する（誤答ありのレコードはこのコースのみ）
- **FR-003**: System MUST 誤答の有無を、ユーザー本人のそのモードにおける全期間の回答履歴から判定する
- **FR-004**: System MUST 卒業後に誤答した場合、従来どおり「覚えている途中」に復帰させ、以後は通常コースを適用する
- **FR-005**: System MUST 既存の同日ガード（正解による前進は1日1回まで）を早期卒業判定にも適用する（同日2回目の正解では卒業しない）
- **FR-006**: System MUST 「誤答履歴がなく正解2回以上」の既存レコードを一括で卒業済みへ移行する手段を提供し、その処理は冪等であること
- **FR-007**: 早期卒業の判定ロジックは自動テストで検証可能であること

### Key Entities

- **復習レコード（srs_records）**: ユーザー×市区町村×モードごとの学習状態（正解回数、復習間隔、状態）。スキーマ変更は行わない
- **回答履歴（municipality_quiz_results）**: 1回答＝1行の正誤記録。誤答有無の判定ソース（既存。正答率表示にも使用中）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 一度も誤答していない市区町村は、最短2日（別日2回の正解）で「覚えている途中」一覧から消える（従来は最短約22日）
- **SC-002**: 機能導入時の一括移行により、「誤答なしで正解2回以上」の既存レコードが全件卒業済みになる
- **SC-003**: 誤答歴のある市区町村の卒業タイミングは従来と変わらない
- **SC-004**: クイズの回答体感速度が悪化しない（回答結果の表示までの時間が現状と同等）

## Assumptions

- まぐれ正解対策として、初回正解だけでは卒業させず別日の2回目正解を確認条件とする（ユーザー確認済み）
- 誤答有無の判定には既存の回答履歴テーブルをそのまま使い、新しいカラムやテーブルは追加しない（正答率表示と同じデータソース。ユーザー確認済み）
- 復習機能（005）導入前の回答履歴も誤答判定の対象に含める
- Mode A の多重INSERTバグ（backlog B00x）由来の重複履歴行は正誤が同一の複製であり、誤答有無の判定には影響しない
- 一括移行は運用者がスクリプトとして手動実行する（アプリ内 UI は提供しない）
