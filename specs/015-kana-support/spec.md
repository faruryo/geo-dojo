# Feature Specification: 都道府県・市区町村の読み仮名表示

**Feature Branch**: `015-kana-support`
**Created**: 2026-07-17
**Status**: Draft
**Input**: User description: "都道府県・市区町村の読み仮名（ふりがな）対応。難読な市区町村の学習効果を高めるため、出題時や結果画面、苦手リスト等で読み仮名（ひらがな）を表示できるようにする。特に、クイズ回答直後の正解・不正解フィードバック（結果表示）のタイミングで読み仮名を表示しておくと、その場で正しい読み方を確認できて学習効果が高い。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 解答直後のフィードバックで正しい読み方がわかる (Priority: P1)

クイズに解答した直後、正解・不正解のフィードバックが表示されるタイミングで、対象の都道府県名・市区町村名の読み仮名（ひらがな）が併記される。これにより、難読な地名でもその場で正しい読み方を確認でき、記憶の定着につながる。

**Why this priority**: ユーザーが最も学習効果が高いと明言している瞬間。これ単体でも「正しい読み方をその場で確認できる」という価値が完結し、最小のリリース単位になる。

**Independent Test**: 任意の出題モードでクイズをプレイして解答し、正解・不正解のフィードバック表示に読み仮名が併記されることを確認すれば、単独で検証・価値提供できる。

**Acceptance Scenarios**:

1. **Given** クイズで問題に解答した、**When** 正解・不正解のフィードバックが表示される、**Then** 対象の都道府県名・市区町村名に読み仮名（ひらがな）が併記される
2. **Given** 難読な市区町村（読み仮名データを持つもの）が出題対象だった、**When** フィードバックを見る、**Then** その市区町村の正しい読み方をその場で確認できる
3. **Given** 読み仮名データが未整備の項目が出題対象だった、**When** フィードバックを見る、**Then** 読み仮名は表示されず、名称表示自体は通常通り行われる

---

### User Story 2 - 苦手リストで読み方を確認できる (Priority: P2)

ダッシュボードや復習画面に表示される苦手市区町村リストの各項目に、市区町村名の読み仮名が併記される。復習対象を眺めるだけで、間違えやすい地名の読み方を再確認できる。

**Why this priority**: P1（フィードバック）を補強し、復習前後でも読み方を確認できる場を増やす。P1単体でも価値は完結するため、これは次点。

**Independent Test**: 苦手市区町村が存在する状態でダッシュボードまたは復習画面の一覧を開き、各項目に読み仮名が併記されていることを確認する。

**Acceptance Scenarios**:

1. **Given** ダッシュボードの苦手市区町村ランキングに項目が表示されている、**When** 一覧を見る、**Then** 各市区町村名に読み仮名が併記されている
2. **Given** 復習対象一覧（復習項目一覧）に項目が表示されている、**When** 一覧を見る、**Then** 各項目の市区町村名に読み仮名が併記されている

---

### （対象外）出題中の問題文・選択肢への読み仮名併記

当初は出題中（解答前）の問題文・選択肢にも読み仮名を併記する案があったが、**スコープ外とした**。ユーザーは読み方そのものを含めて記憶する必要があるため、解答前に読みを見せてしまうと学習効果を損なう。読み仮名は解答後のフィードバック（US1）・苦手リスト（US2）でのみ提供し、出題中は名称（漢字）のみを表示する。

---

### Edge Cases

- **読み仮名データが未整備の都道府県・市区町村**: 該当項目は読み仮名を表示せず、名称表示・レイアウトはそのまま維持する。
- **同じ字面で読みが異なるケースを含む同名の複数コード**（例: 政令指定都市の区が同一市区町村名を共有する既存データ構造）: 各コードは既存の名称と同様、個別に読み仮名を保持するため、名寄せに起因する誤読は発生しない。
- **促音・長音・濁音を含む複雑な読み**: 通常のひらがな表記でそのまま対応し、特別な変換処理は行わない。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは、47都道府県および全市区町村について、読み仮名（ひらがな表記）を保持しなければならない。
- **FR-002**: クイズ回答直後の正解・不正解フィードバック表示において、対象の都道府県名・市区町村名に読み仮名を併記しなければならない。
- **FR-003**: 苦手市区町村リスト（ダッシュボードの苦手市区町村ランキング・復習項目一覧）の各項目に、市区町村名の読み仮名を併記しなければならない。
- **FR-004**: 出題中（解答前）の問題文・選択肢には読み仮名を併記してはならない。ユーザーが読み方を含めて記憶する機会を損なわないため、読み仮名は解答後のフィードバック（FR-002）・苦手リスト（FR-003）でのみ提供する。
- **FR-005**: 読み仮名データが存在しない項目については、読み仮名表示を省略し、名称表示自体やレイアウトを崩してはならない。
- **FR-006**: 読み仮名の併記は、既存の出題・解答・採点・進行（結果集計、復習スケジューリングを含む）の動作に影響を与えてはならない。

### Key Entities *(include if feature involves data)*

- **都道府県（Prefecture）**: 47件。名称に対応する読み仮名（ひらがな）を1件ずつ持つ。
- **市区町村（Municipality）**: 既存の市区町村マスタ内の各レコード（コード単位）。名称に対応する読み仮名（ひらがな）を1件ずつ持つ。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーは、クイズの正解・不正解フィードバックを見た時点で、追加の操作なしにその都道府県・市区町村の読み方を確認できる。
- **SC-002**: 読み仮名データが整備されている都道府県・市区町村について、フィードバック表示・苦手リスト表示のいずれでも読み仮名の表示率が100%になる。
- **SC-003**: 読み仮名表示の追加によって、既存の出題・解答・採点・結果集計・復習スケジューリングのロジックに影響が出ない（既存の自動テストが引き続き通る）。
- **SC-004**: 読み仮名表示の追加により、フィードバック・苦手リストの視認性低下やレイアウト崩れが発生しない。また、出題中（解答前）の問題文・選択肢には読み仮名が表示されないことを維持する。

## Assumptions

- 読み仮名は文字単位のルビ振り（各漢字に対応する個別のふりがな）ではなく、名称全体に対する「ひらがな読み」をまとめて併記する形式とする（例: 「札幌市（さっぽろし）」）。
- 読み仮名表示は常時オンとし、表示・非表示を切り替える専用設定は本機能のスコープ外とする（必要性が見えた場合は別途検討）。
- 読み仮名データは都道府県47件・市区町村全件について一括で整備する。データの出典・収集方法自体は本仕様の対象外とする。
- 政令指定都市の区など同一名称が複数コードにまたがる既存のデータ構造はそのまま踏襲し、読み仮名専用の名寄せや構造変更は行わない。
- 表示位置・スタイルは既存の名称表示に対する補助的な注記として扱い、主たる情報（正解・不正解、選択肢そのもの、地図）の視認性を優先する。
