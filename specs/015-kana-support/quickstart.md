# Quickstart: 都道府県・市区町村の読み仮名表示

## テスト実行

```bash
pnpm test                     # Vitest（kana 伝播ロジック・PREFECTURE_KANA 整合性テストなど）
pnpm lint                     # 型チェック / Lint
```

## 動作確認（ローカル）

### 準備

```bash
supabase start                # ローカル Supabase（起動していない場合のみ）
pnpm drizzle-kit migrate      # municipality_master.kana カラムを反映
pnpm tsx scripts/fetch-municipality-kana.ts    # 総務省データを取り込み seed JSON を生成（初回のみ）
pnpm tsx scripts/import-municipality-kana.ts   # シード JSON から kana を DB へ反映
pnpm dev
```

ブラウザで `http://127.0.0.1:3000` を開く。

> [!NOTE]
> **ログインについて**
> ローカル環境ではメール確認（Email Confirmation）が無効化されています。アカウントが存在しない、あるいはログインできない場合は、新規アカウント作成画面 (`/signup`) から適当なメールアドレスとパスワード（6文字以上）を入力して登録してください。

### 動作確認手順

#### 1. 解答直後のフィードバックでの読み仮名確認（P1）
1. 市区町村クイズ（任意のモード）を開始し、読み仮名データが整備された市区町村（例: 難読地名）が出題されるまでプレイする。
2. 解答した直後の正解・不正解フィードバックに、対象の市区町村名（モードAでは都道府県名も）に読み仮名が併記されていることを確認する。
3. 読み仮名データが未整備の項目が出題された場合、読み仮名は表示されず、名称表示・レイアウトが崩れないことを確認する。

#### 2. 苦手リスト・復習項目一覧での読み仮名確認（P2）
1. いくつか誤答してからダッシュボードを開き、「苦手ランキング」の各項目に読み仮名が併記されていることを確認する。
2. 復習項目一覧画面（`/quiz/review/items`）を開き、各行に読み仮名が併記されていることを確認する。
3. 復習クイズ（`/quiz/review`）をプレイして完了後、「まだ苦手な市区町村」バッジに読み仮名が併記されていることを確認する。

#### 3. 出題中の問題文・選択肢での読み仮名確認（P3）
1. 選択肢形式のモード（都道府県名や市区町村名を文字で選ぶモード）でクイズを開始する。
2. 問題文・各選択肢の地名に読み仮名が併記されていることを確認する。全選択肢に等しく併記され、特定の選択肢だけが強調されていないことを確認する。

#### 4. 既存機能への影響がないことの確認
1. 通常通りクイズをプレイし、採点・結果集計・復習スケジューリング（SM-2）の動作が従来通りであることを確認する。
2. `pnpm test` で既存の自動テストがすべて通ることを確認する。
