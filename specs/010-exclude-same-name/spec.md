# Feature Specification: 都道府県名と同一の市区町村を出題から除外するフィルタ

**Feature Branch**: `010-exclude-same-name`
**Created**: 2026-07-09
**Status**: Superseded — B012（`feat(quiz): exclude same-name municipalities in text modes`, 2026-07-09マージ）が本仕様の中核（FR-001, FR-003, FR-004, FR-006）を先に実装済み。ただし B012 の実装はトグルなしの常時ON（本仕様が求める「デフォルトOFF＋トグル」ではない）。現状の常時ONで十分と判断し、トグルUI（FR-002/FR-005）は見送り、本specはクローズする。
**Input**: User description: "「青森市（青森県）」や「秋田市（秋田県）」などのように、都道府県名と市区町村名（接尾辞を除く）が一致するものは、答えが自明（簡単すぎる）なため、テキストベースのクイズ（Mode Aなど）で出題から除外できるようにする。地図で位置を当てる問題（地図タップ系モードなど）は、名前が一致していても位置当て自体の難易度は下がらないため、除外対象外とする。"

## 背景 / 問題

市区町村クイズにおいて、都道府県名と市区町村名がほぼ同一のもの（例：「青森市」と「青森県」）は、テキスト形式で都道府県名を答えさせたり、都道府県名から市区町村名を選択させたりするモード（Mode A, B, C）では正解が自明（簡単すぎる）である。
これにより、ユーザーにとって学習効果の低い問題がプールに混入し、学習効率が低下する問題がある。
一方、地図タップ形式のクイズ（Mode D: 地図上で市区町村の位置を特定する）では、名前が自明であってもその位置を特定する難易度は下がらないため、出題から除外する必要はない。

## User Scenarios & Testing

### US1 - テキストベースのクイズで同名市区町村を除外する (Priority: P1)
クイズ設定で「同名市区町村を除外」を有効にした場合、Mode A（逆引き地図・テキスト回答）、Mode B（逆引き4択）、Mode C（順引き4択）の出題プールから、都道府県名と市区町村名（接尾辞を除く）が一致する市区町村（例：「青森市」「秋田市」など）が除外される。

**Independent Test**:
Mode B で「同名市区町村を除外」をオンにしてクイズを開始し、「青森市（青森県）」「秋田市（秋田県）」などの自明な問題が出題されないことを検証する。

**Acceptance Scenarios**:
1. **Given** 「同名市区町村を除外」設定が有効, **When** Mode B のクイズを開始する, **Then** 「青森市」「秋田市」などの都道府県名と幹名が一致する市区町村は出題されない
2. **Given** 「同名市区町村を除外」設定が有効, **When** Mode A のクイズを開始する, **Then** 都道府県名と幹名が一致する市区町村は集約・出題されない

---

### US2 - 地図タップモード（Mode D）では除外しない (Priority: P1)
クイズ設定で「同名市区町村を除外」を有効にしていても、Mode D（順引き地図タップ）のクイズでは、同名市区町村が除外されずに出題される。

**Independent Test**:
Mode D で「同名市区町村を除外」をオンにしてクイズを開始し、「青森市（青森県）」や「秋田市（秋田県）」などが出題プールに含まれることを確認する。

**Acceptance Scenarios**:
1. **Given** 「同名市区町村を除外」設定が有効, **When** Mode D のクイズを開始する, **Then** 都道府県名と幹名が一致する市区町村（例：「青森市」）も通常どおり出題される

---

### US3 - 設定のトグルUIと保持 (Priority: P2)
クイズ設定画面（セットアップ画面）に「都道府県名と同一の市区町村を除外」のトグルスイッチを追加する。この設定値は、セッション間またはクイズ開始前の調整ダイアログ（おすすめクイズ）および通常のクイズ設定画面でオン・オフを切り替えることができる。
また、トグル設定値は LocalStorage に保存し、次回アクセス時も維持される。

**Independent Test**:
クイズ設定画面で「都道府県名と同一の市区町村を除外」をオンにしてリロードし、設定がオンのまま保持されていることを確認する。

**Acceptance Scenarios**:
1. **Given** クイズセットアップ画面, **When** 「都道府県名と同一の市区町村を除外」のトグルをオンにする, **Then** トグルの状態が LocalStorage に保存される
2. **Given** 以前にトグルをオフに設定していた場合, **When** ページをリロードする, **Then** トグルの状態がオフとして復元される

---

### Edge Cases

- **「さいたま市（埼玉県）」の扱い**: 「さいたま」と「埼玉」はひらがな・漢字で一致しないため、本フィルタの対象外（除外しない）とする。
- **政令指定都市の区名**: 政令指定都市の「区」はマスタ上で親市名（例：「福岡市」）にマージされているため、「福岡市（福岡県）」は幹名「福岡」同士で一致し、除外対象となる。神奈川区（神奈川県横浜市）などの区レベルはそもそもマスタに存在しないため考慮不要。
- **復習クイズ（/quiz/review）**: 復習クイズは SRS アルゴリズムに基づいて due になった問題を出題するため、この「自明問題除外」フィルタの対象外（常に全問出題）とする。

## Requirements

### Functional Requirements

- **FR-001**: System MUST 都道府県名（末尾の都・道・府・県を除く）と市区町村名（末尾の市・町・村・区を除く）が完全一致するものを「同名市区町村」と判定するロジック（純粋関数）を提供する。
- **FR-002**: System MUST クイズのセットアップ画面（`app/(app)/quiz/municipality/[mode]/page.tsx`）に「都道府県名と同一の市区町村を除外」のトグルスイッチ（デフォルト: オフ）を追加する。
- **FR-003**: System MUST トグルが有効な場合、Mode A, B, C の出題プール（`buildQuestions` 内）から同名市区町村を除外する。
- **FR-004**: System MUST トグルが有効であっても、Mode D の出題プールからは除外しない。
- **FR-005**: System MUST トグルの設定状態を LocalStorage に保存し、ページ遷移やリロード後も設定を維持する。
- **FR-006**: System MUST おすすめクイズの調整ダイアログ（`RecommendOverride`）およびおすすめクイズ開始処理（`getRecommendation` などの推薦エンジン）でも、トグル設定に連動して同名市区町村を除外する（またはクイズ開始時の buildQuestions で一元適用する）。

### Key Entities

- **市区町村データ（Municipality）**: `code`, `name`, `prefecture` を持つオブジェクト。
- **クイズ設定（Settings）**: `excludeSameName: boolean` を追加。

## Success Criteria

### Measurable Outcomes

- **SC-001**: 「同名市区町村を除外」をオンにして Mode B をプレイした際、「青森市（青森県）」「秋田市（秋田県）」などの問題が一切出題されない。
- **SC-002**: 「同名市区町村を除外」をオンにして Mode D をプレイした際、「青森市（青森県）」などの問題が出題される。
- **SC-003**: 設定値が LocalStorage に保存され、ページリロード後もトグル状態が保持される。
