# Feature Specification: ダッシュボード連続記録のモチベーション強化

**Feature Branch**: `004-streak-encouragement`  
**Created**: 2026-05-30  
**Status**: Draft (retroactive — 実装済み変更の事後ドキュメント)  
**Input**: 「ダッシュボードの『今週のベスト（正答率）』カードを廃止し、連続記録カードを単独全幅化したうえで、連続学習日数に応じた称賛・励ましメッセージを表示する」

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 連続日数に応じた称賛・励ましで学習を続けたくなる (Priority: P1)

学習者がダッシュボードを開くと、現在の連続学習日数とあわせて、その記録に見合った称賛または励ましのひと言が表示される。記録を伸ばすほどメッセージが段階的に強くなり、「もう一日続けよう」という気持ちになる。

**Why this priority**: 本機能の中核価値。単なる数値表示では継続動機が弱い。日数に意味づけ（節目・自己記録更新）を与えることで、地理クイズという反復学習の継続率を高める。これ単体で価値が成立する。

**Independent Test**: 連続日数を 0 / 1 / 3 / 5(自己最長更新中) / 8 / 15 / 31 の各値で表示させ、各段階で異なる適切なメッセージが出ることを確認できる。

**Acceptance Scenarios**:

1. **Given** 連続記録が 1 日, **When** ダッシュボードを開く, **Then** 「いいスタート」相当の前向きなメッセージが表示される
2. **Given** 連続記録が 7 日（節目）, **When** ダッシュボードを開く, **Then** 「1週間達成」を強調する称賛メッセージが目立つ色で表示される
3. **Given** 連続記録が 5 日かつそれが自己最長, **When** ダッシュボードを開く, **Then** 自己最長更新中であることを称えるメッセージが表示される
4. **Given** 連続記録が 0 日, **When** ダッシュボードを開く, **Then** 称賛メッセージは表示されず、学習再開を促す導線のみが表示される

---

### User Story 2 - 誤解を招く正答率指標が整理され、ダッシュボードが正直になる (Priority: P2)

学習者は「今週のベスト正答率」のような、難易度・問題数を無視した正答率表示や、互いに矛盾するメッセージ（「自己ベスト更新！」と「前回比 −30%」の同居）を目にしなくなる。残る正答率指標は重複なく一貫している。

**Why this priority**: 既存カードは正答率を 3 箇所で重複表示し、かつ算出対象の異なる 2 つの「ベスト」を 1 枚に混在させて矛盾した印象を与えていた。これは指標としての信頼性を損なう。P1 の体験を素直に見せる前提整理。

**Independent Test**: ダッシュボード上の正答率表示箇所を数え、廃止前後で重複が減っていること、矛盾する同時表示が発生しないことを確認できる。

**Acceptance Scenarios**:

1. **Given** 今週に複数セッションを実施した学習者, **When** ダッシュボードを開く, **Then** 「週内ベスト正答率」カードは表示されない
2. **Given** 直近セッションが前回より低い正答率だった, **When** ダッシュボードを開く, **Then** 「自己ベスト更新」と「前回比マイナス」が同時に表示されることはない

---

### User Story 3 - 連続記録カードが見やすくなる (Priority: P3)

連続記録は半分幅の窮屈な並びではなく、単独で全幅表示され、数値とメッセージが読みやすい。

**Why this priority**: 視認性の改善。P1 のメッセージを置くスペースも確保する。価値はあるが単独では補助的。

**Acceptance Scenarios**:

1. **Given** ダッシュボード（モバイル幅 375px）, **When** 連続記録セクションを見る, **Then** 連続記録カードが横並びの相方なしに全幅で表示される

---

### Edge Cases

- **連続 0 日（途切れた直後 / 未開始）**: 称賛は出さず、再開を促す導線のみ。空欄や不自然な「0日おめでとう」を出さない。
- **自己最長を更新中（節目未満）**: 7/14/30 日の大きな節目に達していなくても、現在記録が過去最長と同値以上なら、その達成を称える。
- **大きな節目と自己最長更新の重複**: 節目（1週間/2週間/1ヶ月）のメッセージを優先し、節目未満の範囲でのみ「自己最長更新中」を表示する。
- **初日で最長記録も 1**: 初日相当の「いいスタート」メッセージとして扱い、過剰な称賛を避ける。
- **今日まだ未学習**: 連続記録の有無にかかわらず、本日分の学習を促す導線を併記する。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは現在の連続学習日数と過去最長の連続日数を常に表示しなければならない。
- **FR-002**: システムは連続日数が 1 日以上のとき、その値に応じた称賛または励ましメッセージを 1 件表示しなければならない。
- **FR-003**: システムは 1 週間・2 週間・1 ヶ月といった大きな節目に達したとき、メッセージを視覚的に強調（目立つアクセント表示）しなければならない。
- **FR-004**: システムは大きな節目未満であっても、現在の連続日数が過去最長以上のとき、自己最長を更新中であることを称えるメッセージを表示しなければならない。
- **FR-005**: システムは連続日数が 2 段階以上の強さで段階的に変化するメッセージを提供しなければならない（最低 4 段階：開始直後 / 習慣化途上 / 自己記録更新 / 大きな節目）。
- **FR-006**: システムは連続日数が 0 のとき、称賛メッセージを表示せず、学習再開を促す導線のみを表示しなければならない。
- **FR-007**: システムは本日まだ学習していない場合、本日分の学習を開始できる導線を表示しなければならない。
- **FR-008**: システムはダッシュボードから「週内ベスト正答率」カードを廃止し、正答率の重複表示および矛盾する同時メッセージ（自己ベスト更新と前回比マイナスの同居）を発生させてはならない。
- **FR-009**: システムは連続記録カードを横並びの相方なしに単独全幅で表示しなければならない。
- **FR-010**: メッセージ判定はデータ保存層に変更を加えず、既存の連続記録データのみから決定論的に算出されなければならない。

### Key Entities *(include if feature involves data)*

- **連続記録 (Streak)**: 学習者の学習継続状態を表す。属性: 現在の連続日数、過去最長の連続日数、本日学習済みか。新規データ項目は追加しない（既存の算出値を再利用）。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 学習者は連続日数を見た瞬間に「自分が今どの段階にいるか（開始/継続中/記録更新中/節目達成）」を、補足説明なしに理解できる。
- **SC-002**: ダッシュボード上で互いに矛盾する評価メッセージ（称賛と否定的比較の同居）が表示される箇所が 0 件になる。
- **SC-003**: 正答率の表示箇所が 3 箇所から 2 箇所に削減され、同一指標の重複がなくなる。
- **SC-004**: 連続日数に応じたメッセージが最低 4 段階で段階的に変化し、記録を伸ばすほど称賛が強くなる。
- **SC-005**: 連続記録カードはモバイル幅（375px）でも縦スクロールのみで全内容が読め、メッセージが折り返しで崩れない。

## Assumptions

- 連続日数・最長日数・本日学習済み判定は、既存の連続記録算出ロジックをそのまま再利用する（本機能で算出方法は変更しない）。
- 正答率という指標自体は引き続き別カード（累計サマリー・正答率推移）に残るため、「週内ベスト正答率」カードの廃止による情報損失はない。
- メッセージは日本語固定、ダークモード前提の表示とする。
- メッセージのしきい値（1 日 / 2 日 / 4 日 / 7 日 / 14 日 / 30 日、および自己最長更新）は本仕様の既定値とし、文言・しきい値の調整は将来の改修で容易に変更できることを前提とする。
- 本機能はダッシュボード画面のみを対象とし、クイズ実行画面や通知（プッシュ）は対象外（将来の拡張候補）。
